# 动画系统概览

## 概述

Lyra 的动画系统采用**分层动画架构**，通过 AnimBlueprint 的 LayeredBoneBlend 实现不同身体部位（上半身/下半身/头部）的独立动画控制。结合动态动画节点和 Animation Sharing，支持多人同屏的高效动画播放。

---

## 动画架构层级

```
ULyraAnimInstance（动画蓝图实例）
    │  路径：Source/LyraGame/Animation/LyraAnimInstance.h
    │
    ├── 基础姿势层（Idle/Jog/Sprint）
    │   └── 全身骨骼，受移动状态影响
    │
    ├── 上半身覆盖层（Aim/Shoot/Reload）
    │   └── Blend in 到 spine/upperarm/hand
    │
    └── 头部覆盖层（LookAt/TurnInPlace）
        └── Blend in 到 neck/head
```

---

## 分层混合机制

Lyra 使用 `LayeredBoneBlend` 节点实现分层控制：

```
LayeredBoneBlend（PerBoneBlendMode）
    │
    ├── Layer 0: 全身基础动画（移动、跳跃）
    │   └── BoneMask: 全身
    │
    ├── Layer 1: 上半身动作（瞄准、射击、换弹）
    │   └── BoneMask: Spine → UpperArm → Hand
    │       BlendIn: 0.1s, BlendOut: 0.2s
    │
    └── Layer 2: 头部动作（观察方向）
        └── BoneMask: Neck → Head
            BlendIn: 0.05s, BlendOut: 0.15s
```

这种架构允许角色在跑动时上半身独立执行射击动画，下半身保持移动动画。

---

## 动态动画节点

Lyra 实现了运行时可切换的动画节点，通过数据驱动动画选择：

```
// 伪代码示意 — Lyra 使用自定义 AnimNode
AnimNode_LayeredBoneBlendPerBone
    → 运行时根据 GameplayTag 切换动画集
    → 不同武器的持枪姿势通过动画集切换
    → 不需为每种武器创建独立的 AnimBlueprint
```

武器类型与动画集的对应关系：
```
装备武器 → EquipmentManagerComponent 通知 AnimInstance
    → AnimInstance 切换动画集（IdlePose / AimPose / FirePose）
    → LayeredBoneBlend 各层切换到新动画集
```

---

## Animation Sharing

Lyra 集成了 UE 的 Animation Sharing 系统，优化同屏大量角色的动画性能：

```
AnimationSharingManager
    ├── 同类角色共享 AnimationInstance
    ├── 同帧播放相同动画序列（硬件实例化）
    └── 显著降低同屏多人时的动画 CPU 开销
```

触发条件：
- 非玩家控制的角色（AI 敌人）
- 距离玩家较远的角色
- 使用相同动画蓝图的角色

## 动画蓝图事件

LyraAnimInstance 处理的关键事件：

| 事件 | 触发条件 | 典型行为 |
|------|---------|---------|
| OnMontageStarted | Montage 开始播放 | 切换 LayeredBoneBlend 权重 |
| OnMontageBlendingOut | Montage 淡出 | 恢复到默认动画状态 |
| OnMontageEnded | Montage 播放完成 | 清理动画状态 |
| OnWeaponEquipped | 装备切换 | 更新武器动画集 |

---

## 关键设计要点

1. **分层混合** — 身体不同部位独立控制，支持上半身射击 + 下半身移动的组合
2. **动画集切换** — 武器/装备切换时只更新动画集，不切换整个 AnimBlueprint
3. **Animation Sharing** — 非玩家角色共享动画实例，降低多人场景的动画开销
4. **Tag 驱动** — 动画状态通过 GameplayTag 驱动，与 GAS/Equipment 的 Tag 体系一致
