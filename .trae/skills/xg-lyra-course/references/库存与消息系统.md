# 库存与消息系统

## 概述

库存系统与消息系统共同构成了 Lyra 中**物品管理**和**事件通信**的基础设施。库存系统管理物品的定义、存储和消耗；消息系统提供跨系统的松耦合通信机制；游戏阶段系统则利用 GAS 驱动游戏流程的阶段切换。

---

## 库存系统：四层架构

**文件**：`Source/LyraGame/Inventory/LyraInventoryManagerComponent.h`

库存系统分为四个层级：

```
Item Definition Layer（定义层）
  ULyraInventoryItemDefinition —— 物品模板（UObject, Blueprintable, Abstract）
    └── Fragments[] —— 物品片段数组（EditInlineNew）

Item Instance Layer（运行时实例层）
  ULyraInventoryItemInstance —— 运行时物品实例（Replicated）
    ├── ItemDef —— 指向定义
    └── StatTags —— 统计标签堆栈

Inventory List Layer（清单层）
  FLyraInventoryList —— FFastArraySerializer
    └── Entries[] —— FLyraInventoryEntry
          ├── Instance -> ULyraInventoryItemInstance
          └── StackCount —— 堆叠数量

Inventory Manager Layer（管理器层）
  ULyraInventoryManagerComponent —— UActorComponent
    └── InventoryList —— 物品清单
```

### 定义层：ULyraInventoryItemDefinition

**文件**：`Source/LyraGame/Inventory/LyraInventoryItemDefinition.h`

```cpp
UCLASS(Abstract, Blueprintable)
class ULyraInventoryItemDefinition : public UObject
{
    GENERATED_BODY()

public:
    UPROPERTY(EditDefaultsOnly, Category = "Inventory")
    FText DisplayName;

    // 物品片段，EditInlineNew 可在蓝图编辑器中直接配置
    UPROPERTY(EditDefaultsOnly, Instanced, Category = "Inventory")
    TArray<TObjectPtr<ULyraInventoryItemFragment>> Fragments;
};
```

### Fragment 系统

`ULyraInventoryItemFragment` 是 `EditInlineNew` 抽象基类，通过组合模式扩展物品功能：

```cpp
UCLASS(Abstract, DefaultToInstanced, EditInlineNew)
class ULyraInventoryItemFragment : public UObject
{
    GENERATED_BODY()

public:
    // 实例创建时回调
    virtual void OnInstanceCreated(ULyraInventoryItemInstance* Instance) const;
};
```

在物品定义 Blueprint 中添加 Fragment 子类来扩展功能，如武器段添加伤害属性、消耗品段添加回复效果。

### 实例层：ULyraInventoryItemInstance

### 清单层：FLyraInventoryList

继承 `FFastArraySerializer`，实现网络增量复制：

```
FLyraInventoryEntry
  ├── Instance : ULyraInventoryItemInstance（Replicated）
  └── StackCount : int32（Replicated）

FLyraInventoryList
  ├── GetAllItems()
  ├── AddEntry(ItemClass, StackCount)
  ├── AddEntry(Instance)
  └── RemoveEntry(Instance)
```

### 管理器层：ULyraInventoryManagerComponent

主要 API 入口：

| 方法 | 说明 |
|------|------|
| `CanAddItemDefinition(ItemDef, StackCount)` | 检查能否添加物品 |
| `AddItemDefinition(ItemDef, StackCount)` | 按定义添加物品，返回实例 |
| `AddItemInstance(Instance)` | 直接添加实例 |
| `RemoveItemInstance(Instance)` | 移除实例 |
| `GetAllItems()` | 获取所有实例列表 |
| `ConsumeItemsByDefinition(ItemDef, NumToConsume)` | 消耗指定数量的物品 |

### ALyraWeaponSpawner：武器生成器

**文件**：`Source/LyraGame/Weapons/LyraWeaponSpawner.h`

场景中生成可拾取武器的 Actor，支持拾取—冷却—重生循环：

```cpp
UCLASS()
class ALyraWeaponSpawner : public AActor
{
    ULyraWeaponPickupDefinition* WeaponDefinition;
    bool bIsWeaponAvailable;              // Replicated
    float CoolDownTime;
    float CheckExistingOverlapDelay;

    UCapsuleComponent* CollisionVolume;
    UStaticMeshComponent* PadMesh;
    UStaticMeshComponent* WeaponMesh;
};
```

#### 拾取—冷却—重生流程

```
OnOverlapBegin（玩家进入碰撞体）
  → 检查 bIsWeaponAvailable
  → AttemptPickUpWeapon(Pawn)
    → GiveWeapon(WeaponItemClass, Pawn)（BlueprintImplementableEvent）
    → 创建 InventoryItemInstance 添加到玩家背包
  → StartCoolDown()
    → SetWeaponPickupVisibility(false)
    → bIsWeaponAvailable = false
    → 启动冷却定时器

冷却完成
  → bIsWeaponAvailable = true
  → SetWeaponPickupVisibility(true)
  → PlayRespawnEffects()（BlueprintNativeEvent）
  → CheckForExistingOverlaps()（检测底座上是否有等待的 Pawn）
```

#### ULyraPickupDefinition / ULyraWeaponPickupDefinition

DataAsset 定义拾取物属性：

| 字段 | 类型 | 说明 |
|------|------|------|
| `InventoryItemDefinition` | TSubclassOf | 拾取后添加的物品定义 |
| `DisplayMesh` | UStaticMesh | 拾取物显示网格 |
| `SpawnCoolDownSeconds` | float | 重生冷却时间 |

武器拾取定义额外包含 `WeaponMeshOffset` 和 `WeaponMeshScale`。

## GameplayMessageRouter：消息路由系统

### 插件层

**文件**：`Plugins/GameplayMessageRouter/Source/GameplayMessageRuntime/Public/GameFramework/GameplayMessageSubsystem.h`

作为 `UGameInstanceSubsystem`，提供基于 GameplayTag 的消息发布—订阅机制：

```cpp
class UGameplayMessageSubsystem : public UGameInstanceSubsystem
{
    // 广播消息，支持父标签遍历
    void BroadcastMessage(FGameplayTag Tag, const TSharedPtr<void>& Message);

    // 注册监听器
    void RegisterListener(FGameplayMessageListenerParams& Params);

    // 蓝图版本
    UFUNCTION(BlueprintCallable)
    void K2_BroadcastMessage(FGameplayTag Tag, const int32& Message);
};
```

`FGameplayMessageListenerParams` 配置参数：

```cpp
struct FGameplayMessageListenerParams
{
    FGameplayTag Tag;                                                  // 监听的标签
    TFunction<void(FGameplayTag, const TSharedPtr<void>&)> Callback;   // 回调
    EGameplayMessageMatch MatchType;                                   // ExactMatch / PartialMatch
};
```

### 应用层：FLyraVerbMessage

**文件**：`Source/LyraGame/Messages/LyraVerbMessage.h`

核心消息结构，描述"谁(Instigator) 对谁(Target) 做了什么(Verb)"：

```cpp
USTRUCT(BlueprintType)
struct FLyraVerbMessage
{
    FGameplayTag Verb;                    // 动作标签
    TObjectPtr<UObject> Instigator;       // 发起者
    TObjectPtr<UObject> Target;           // 目标
    FGameplayTagContainer InstigatorTags; // 发起者标签
    FGameplayTagContainer TargetTags;     // 目标标签
    FGameplayTagContainer ContextTags;    // 上下文标签
    double Magnitude;                     // 强度值
};
```

### FLyraVerbMessageReplication：消息复制

**文件**：`Source/LyraGame/Messages/LyraVerbMessageReplication.h`

基于 `FFastArraySerializer` 的服务器到客户端消息复制机制：

```
服务器调用 AddMessage()
  → 消息加入 CurrentMessages 数组，标记为脏
  → FastArray 通过 NetDeltaSerialize 自动同步到客户端
  → 客户端 PostReplicatedAdd / PostReplicatedChange
    → 通过 UGameplayMessageSubsystem 本地重新广播
    → 本地监听者收到消息
```

### FLyraNotificationMessage：通知消息

**文件**：`Source/LyraGame/Messages/LyraNotificationMessage.h`

用于击杀反馈、拾取提示等 UI 瞬时通知：

### UGameplayMessageProcessor：消息处理器基类

**文件**：`Source/LyraGame/Messages/GameplayMessageProcessor.h`


## 游戏阶段系统

### 架构概览

游戏阶段系统使用 GAS 能力驱动游戏流程的阶段切换，支持阶段嵌套、观察者模式和阶段互斥。

### ULyraGamePhaseAbility：阶段能力

**文件**：`Source/LyraGame/AbilitySystem/Phases/LyraGamePhaseAbility.h`

```cpp
UCLASS(Abstract, HideCategories = Input)
class ULyraGamePhaseAbility : public ULyraGameplayAbility
{
    GENERATED_BODY()

public:
    // 此阶段绑定的 GameplayTag
    UPROPERTY(EditDefaultsOnly, BlueprintReadOnly, Category = "Lyra|Game Phase")
    FGameplayTag GamePhaseTag;

    const FGameplayTag& GetGamePhaseTag() const { return GamePhaseTag; }
};
```

Tag 的层级结构决定阶段的父子关系。编辑器颜色为蓝绿色 `FColor(0, 96, 128)`。

### ULyraGamePhaseSubsystem：阶段子系统

**文件**：`Source/LyraGame/AbilitySystem/Phases/LyraGamePhaseSubsystem.h`

```cpp
class ULyraGamePhaseSubsystem : public UWorldSubsystem
{
    // 激活阶段
    void StartPhase(TSubclassOf<ULyraGamePhaseAbility> PhaseAbility,
                    FLyraGamePhaseDelegate PhaseEndedCallback);

    // 观察阶段开始（如果已激活立即调用）
    void WhenPhaseStartsOrIsActive(FGameplayTag PhaseTag, EPhaseTagMatchType MatchType,
                                   const FLyraGamePhaseTagDelegate& WhenPhaseActive);

    // 观察阶段结束
    void WhenPhaseEnds(FGameplayTag PhaseTag, EPhaseTagMatchType MatchType,
                       const FLyraGamePhaseTagDelegate& WhenPhaseEnd);

    // 阶段是否激活
    bool IsPhaseActive(const FGameplayTag& PhaseTag) const;

private:
    TMap<FGameplayAbilitySpecHandle, FLyraGamePhaseEntry> ActivePhaseMap;
    TArray<FPhaseObserver> PhaseStartObservers;
    TArray<FPhaseObserver> PhaseEndObservers;
};
```

### 激活流程

```
蓝图调用 K2_StartPhase
  → ULyraGamePhaseSubsystem::StartPhase
  → 获取 GameState 的 ASC
  → GiveAbilityAndActivateOnce（GA 激活）
  → GA::ActivateAbility
    → PhaseSubsystem::OnBeginPhase
    → 互斥检查
```

### 阶段互斥逻辑

```
当前激活 Game.A
- 新阶段 Game.A.1 → Game.A 保持激活（父阶段），Game.A.1 激活（子阶段）
- 新阶段 Game.B   → Game.A 被取消（旁支兄弟）
- 新阶段 Game.A.2 → Game.A 保持激活，Game.A.1 被取消（同级替换）
```

核心判断：`!IncomingPhaseTag.MatchesTag(ActivePhaseTag)` → 不是子阶段关系时取消当前阶段。

### 典型阶段示例

| 阶段 | 行为 |
|------|------|
| 暖场阶段 | 施加全局无敌 GE_PregameLobby，播放等待音效，3 秒倒计时 |
| 战斗阶段 | 激活人物 Reset 能力，清理场景手榴弹和武器生成器 |
| 计分阶段 | 全局无敌禁止技能，延迟后 ServerTravel 下一场 |

## 关键设计要点

1. **库存四层架构** — 定义层 / 实例层 / 清单层 / 管理器层职责分离，Fragment 系统实现组合式扩展
2. **FastArray 网络同步** — FLyraInventoryList 和 FLyraVerbMessageReplication 均使用 FFastArraySerializer 增量复制
3. **IPickupable 接口** — 统一可拾取物品的契约，支持 Template（按定义生成）和 Instance（直接引用）两种拾取方式
4. **消息系统双层设计** — 插件层提供通用 Tag 消息路由，应用层基于 VerbMessage 构建领域协议
5. **消息复制解耦** — 服务器通过 FastArray 复制原始消息，客户端通过本地消息系统重新广播
6. **GamePhase 基于 GAS** — 阶段切换通过 GA 驱动，Tag 层级天然支持阶段嵌套和互斥
7. **观察者模式** — WhenPhaseStartsOrIsActive / WhenPhaseEnds 支持在阶段变更时执行自定义逻辑
