# 装备与武器系统

## 概述

Lyra 的装备系统采用**三层架构**，将装备的定义、运行时实例、管理器分离，并通过 AbilitySet 机制将装备的能力授予与回收自动化。

---

## 三层架构

```
ULyraEquipmentDefinition（定义层）
    │  数据资产，描述装备的类型和关联 AbilitySet
    │  只存放配置，无运行时状态
    │
    ▼
ULyraEquipmentInstance（实例层）
    │  运行时对象，持有 SpawnedActor 和 GrantedAbilitySet
    │  每个装备实例独立存在
    │
    ▼
ULyraEquipmentManagerComponent（管理层）
    │  管理装备列表（FastArray 网络复制）
    │  提供装备/卸装接口
```

## 第二层：EquipmentInstance

**文件**：`Source/LyraGame/Equipment/LyraEquipmentInstance.h`

```cpp
UCLASS(BlueprintType)
class ULyraEquipmentInstance : public UObject
{
    GENERATED_BODY()

public:
    // 获取装备所属的 Pawn
    UFUNCTION(BlueprintPure)
    APawn* GetTypedPawn() const;

    // 获取装备的 Actor（如武器模型）
    UFUNCTION(BlueprintPure)
    AActor* GetSpawnedActor() const;

    // 装备到指定 Pawn
    virtual void OnEquipped(APawn* Owner);

    // 从 Pawn 卸装
    virtual void OnUnequipped();

    // 获取此装备授予的 AbilitySet Handles（用于卸装时回收）
    const TArray<FGameplayAbilitySpecHandle>& GetGrantedAbilityHandles() const
    {
        return GrantedAbilityHandles;
    }

protected:
    // 生成的 Actor（如武器网格体）
    UPROPERTY()
    TObjectPtr<AActor> SpawnedActor;

    // AbilitySet 授予时记录的 Handle
    TArray<FGameplayAbilitySpecHandle> GrantedAbilityHandles;
};
```

## 装备切换流程

```
ULyraQuickBarComponent::SetActiveSlot(NewSlot)
    │
    ├── 卸装旧装备
    │   → EquipmentManagerComponent::RemoveEquipment(OldEquip)
    │   → OldEquip::OnUnequipped()
    │   → ASC::RemoveAbility(OldEquip.GetGrantedAbilityHandles())
    │   → OldEquip 标记为待回收
    │
    └── 装备新装备
        → EquipmentManagerComponent::AddEquipment(DefinitionClass)
        → NewEquip 创建
        → NewEquip::OnEquipped(OwnerPawn)
        → SpawnEquipmentActor (生成武器网格体)
        → Definition.AbilitySetsToGrant → ASC::GiveAbility
        → 记录 GrantedAbilityHandles 到 NewEquip
```

## Spread/Heat 扩散系统

### 核心接口

**文件**：`Source/LyraGame/AbilitySystem/LyraAbilitySourceInterface.h`

```cpp
struct ILyraAbilitySourceInterface
{
    virtual float GetDistanceAttenuation() const = 0;
    virtual float GetSpreadAngleMultiplier() const = 0;
};
```

### 扩散角度计算

```
当前扩散角度 = Heat × SpreadAngleCurve(Heat)

其中 SpreadAngleCurve 是一个 UCurveFloat，Heat → SpreadAngle 映射
```

### 精度乘数系统

最终着弹散布由以下 4 个乘数共同决定：

| 乘数 | 说明 |
|------|------|
| SpreadRateMultiplier | Heat 增长速率 |
| SpreadAngleMultiplier | 基础扩散角度缩放 |
| StandStillSpreadMultiplier | 站立不动时的精度补偿 |
| CrouchingSpreadMultiplier | 蹲下时的精度补偿 |

```
最终散布角度 = BaseSpread × SpreadAngleMultiplier × StandStillMultiplier × CrouchingMultiplier
```

## SourceObject 回溯

EquipmentInstance 作为 GA 的 SourceObject，使得 GA 可以回溯到授予它的装备：

```
GA 内调用 GetSourceObject()
    → 返回 ULyraEquipmentInstance
    → EquipmentInstance.GetEquipmentDefinition()
    → 返回 ULyraEquipmentDefinition（数据资产配置）
    → 读取武器范围、伤害类型等配置数据
```

这允许同一个 GA 被多种武器复用，通过 SourceObject 区分不同武器的参数。

---

## 关键设计要点

1. **三层职责分离** — Definition 不可变配置 / Instance 运行时状态 / Manager 网络同步
2. **AbilitySet 自动化** — 装备时自动授予，卸装时自动回收，不需要手动管理 GA 生命周期
3. **FastArray 复制** — 装备列表通过 FFastArraySerializer 复制，只同步增量和变更
4. **SourceObject 回溯** — 运行时 GA 可通过 SourceObject 获取装备配置，无需硬编码判断
5. **Spread/Heat 曲线化** — 扩散行为通过曲线和乘数组合，可通过数据配置实现不同手感
