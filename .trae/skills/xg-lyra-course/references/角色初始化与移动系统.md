# 角色初始化与移动系统

## 概述

本章涵盖角色从"被生成"到"可控制"的完整初始化流程，以及移动系统的关键扩展。

---

## PlayerSpawningManager：玩家生成管理

**文件**：`Source/LyraGame/Player/LyraPlayerSpawningManagerComponent.h`

`ULyraPlayerSpawningManagerComponent` 继承自 `UGameStateComponent`，挂载在 `ALyraGameState` 上，统一管理玩家生成点的分配。

```cpp
UCLASS()
class ULyraPlayerSpawningManagerComponent : public UGameStateComponent
{
    GENERATED_BODY()

public:
    virtual AActor* OnChoosePlayerStart(AController* Player, TArray<APlayerStart*>& Starts);
    APlayerStart* GetBestPlayerStart(AController* Player);
    virtual void InitializeStarts();

protected:
    TArray<APlayerStart*> PlayerStarts;
    TSet<APlayerStart*> OccupiedStarts;
};
```

### 生成流程

```
ALyraGameMode::HandleStartingNewPlayer
  → ULyraPlayerSpawningManagerComponent::OnChoosePlayerStart
    → 从 PlayerStarts 中选择合适的起点
    → 考虑：队伍归属、是否被占用、优先级
  → SpawnDefaultPawnAtTransform
    → 创建 ALyraPawn
    → 初始化 PawnExtensionComponent（InitState 状态机启动）
  → Possess Pawn
    → PlayerController 控制 Pawn
```

### 队伍生成逻辑

- 从 `PlayerStarts` 中筛选匹配队伍标签的起点
- 同一队伍的起点分区布置
- `PlayerStartTag` 用于标记特定队伍的起始点

## 移动系统与加速度同步

**文件**：`Source/LyraGame/Character/LyraCharacterMovementComponent.h`

`ULyraCharacterMovementComponent` 继承自 `UCharacterMovementComponent`，由 `ALyraCharacter` 通过 `SetDefaultSubobjectClass` 在构造函数中注入。

### 加速度复制（FLyraReplicatedAcceleration）

Lyra 将加速度矢量从 Cartesian 坐标量化为 Polar 坐标，减少网络带宽占用：

```cpp
struct FLyraReplicatedAcceleration {
    uint8 AccelXYRadians;    // 水平方向角 [0, 255]，由 Cartesian→Polar 量化
    uint8 AccelXYMagnitude;  // 水平大小 [0, 255]
    int8  AccelZ;            // 垂直分量 [-127, 127]
};
```

复制条件仅针对模拟客户端：

```
服务端
  ALyraCharacter::PreReplication()
    → CartesianToPolar(FVector) → 量化 → FLyraReplicatedAcceleration

复制条件
  DOREPLIFETIME_CONDITION(ReplicatedAcceleration, COND_SimulatedOnly)

客户端
  ALyraCharacter::OnRep_ReplicatedAcceleration()
    → PolarToCartesian → FVector
    → ULyraCharacterMovementComponent::SetReplicatedAcceleration(FVector)
```

### SimulateMovement 覆盖

```cpp
void ULyraCharacterMovementComponent::SimulateMovement(float DeltaTime)
{
    if (bHasReplicatedAcceleration)
    {
        Acceleration = ReplicatedAcceleration;
    }
    Super::SimulateMovement(DeltaTime);
}
```

### 地面检测（FLyraCharacterGroundInfo）

按帧缓存的地面检测结果，同一帧内重复调用不重复 Trace：

```cpp
struct FLyraCharacterGroundInfo {
    uint64 LastUpdateFrame;        // 缓存的帧计数
    FHitResult GroundHitResult;    // 检测命中结果
    float GroundDistance;          // 离地距离
};
```

- MOVE_Walking 模式下 GroundDistance = 0
- 其他模式从胶囊中心减去半高向下做线检测

### Tag 控制运动

`ULyraCharacterMovementComponent` 通过 GameplayTag 控制角色运动：

```cpp
FRotator ULyraCharacterMovementComponent::GetDeltaRotation(float DeltaTime) const
{
    if (HasMatchingGameplayTag(TAG_Gameplay_MovementStopped))
        return FRotator(0, 0, 0);  // 不旋转
    return Super::GetDeltaRotation(DeltaTime);
}

float ULyraCharacterMovementComponent::GetMaxSpeed() const
{
    if (HasMatchingGameplayTag(TAG_Gameplay_MovementStopped))
        return 0;  // 不动
    return Super::GetMaxSpeed();
}
```

### Crouched Jump

`CanAttemptJump()` 移除了父类中的 `IsCrouched()` 检查，允许角色在蹲伏状态下跳跃。

### FastShared 复制

Lyra 使用 FastShared 复制模式为模拟客户端提供高频位置更新，详见角色类层级。

## 关键设计要点

1. **PlayerSpawningManager 统一管理** — 替代传统的 AGameModeBase::ChoosePlayerStart，支持队伍分区和冲突避免
2. **加速度 Polar 量化** — Cartesian→Polar 压缩减少带宽，COND_SimulatedOnly 仅同步给模拟客户端
3. **地面检测帧缓存** — FLyraCharacterGroundInfo 按帧缓存，避免单帧多次 Trace
4. **Tag 控制运动** — 通过 GameplayTag 直接控制旋转/速度，无需硬编码状态判断
5. **Crouched Jump** — 移除蹲伏跳跃限制，增加移动灵活性
