# 武器准心与命中标记系统

## 概述

武器准心系统由三个层次组成：**WeaponStateComponent**（追踪命中状态）、**ReticleWidget**（准心扩散渲染）、**HitMarkerConfirmationWidget**（命中反馈标记）。

## 武器状态组件（ULyraWeaponStateComponent）

[LyraWeaponStateComponent.h](file:///d:/UPS/GitLab/XGUnrealNote/Courses/Lyra-Deep-Dive/code/Lyra/Source/LyraGame/Weapons/LyraWeaponStateComponent.h)

### 核心数据

```cpp
struct FLyraScreenSpaceHitLocation
{
    FVector2D Location;       // 视口屏幕空间坐标
    FGameplayTag HitZone;     // 命中区域（TAG_Gameplay_Zone）
    bool bShowAsSuccess;      // 是否成功命中
};

struct FLyraServerSideHitMarkerBatch
{
    TArray<FLyraScreenSpaceHitLocation> Markers;
    uint8 UniqueId;           // 批次唯一 ID
};
```

### AddUnconfirmedServerSideHitMarkers

1. 遍历 FoundHits 中的每个 FHitResult
2. 将世界坐标投影到屏幕空间
3. 通过 `ShouldShowHitAsSuccess()` 判断是否成功命中：
   - 默认：命中不同队伍的目标为成功
4. 通过 HitResult 的 PhysMatWithTags 检查 `TAG_Gameplay_Zone` 标签，确定 HitZone
5. 创建 `FLyraScreenSpaceHitLocation` 并加入批次

### ClientConfirmTargetData（Client, Reliable）

服务端调用此 RPC 确认命中：
1. 根据 UniqueId 匹配未确认的 MarkerBatch
2. 如果 `bSuccess == true` 且 `HitReplaces.Num() != Markers.Num()`：
   - 将未被替换的命中标记（bShowAsSuccess 为 true）添加到 `LastWeaponDamageScreenLocations`
3. 从 `UnconfirmedServerSideHitMarkers` 中移除该批次

## 准心控件（ULyraReticleWidgetBase）

```cpp
UCLASS()
class ULyraReticleWidgetBase : public UUserWidget
{
    // 从关联的 RangedWeaponInstance 读取扩散角
    float ComputeSpreadAngle() const;
    // 计算屏幕空间最大扩散半径（投影数学）
    float ComputeMaxScreenspaceSpreadRadius() const;
    // 是否有首发射击精度

```

- `ComputeSpreadAngle()` — 从 `ULyraRangedWeaponInstance->GetCalculatedSpreadAngle()` 获取当前扩散角度
- `ComputeMaxScreenspaceSpreadRadius()` — 将扩散角度转换为屏幕空间像素偏移
- `InitializeFromWeapon()` — 通过 WeaponInstance 的 Instigator 获取 InventoryInstance

### SCircumferenceMarkerWidget（SLeafWidget）

内部 Slate 控件，在 `OnPaint` 中沿圆形排列绘制标记图像：
- 根据 PositionAngle 计算每个标记的位置
- 应用 ImageRotationAngle 的渲染变换
- 支持 ColorAndOpacity 颜色和透明度

## 命中标记确认控件（SHitMarkerConfirmationWidget）

```cpp
class SHitMarkerConfirmationWidget : public SLeafWidget
{
    FSlateBrush PerHitMarkerImage;          // 每个命中标记的图像
    FSlateBrush AnyHitsMarkerImage;         // 任意命中的图像
    float HitNotifyDuration;                // 通知持续时长（含淡出）
    TMap<FGameplayTag, FSlateBrush> PerHitMarkerZoneOverrideImages;  // 按区域（Zone）覆盖标记图像

    // Tick 中更新 HitNotifyOpacity
};
```

- **PerHitMarkerImage** — 每个命中位置的标记图像
- **AnyHitsMarkerImage** — 只要有命中就显示的标记图像
- **PerHitMarkerZoneOverrideImages** — 按命中区域（如头部/TAG_Gameplay_Zone_Head）覆盖标记图像
- **淡出效果**：HitNotifyDuration 结束后透明度逐步降低

## 整体数据流

```
开火 → PerformLocalTargeting → 命中结果
  → AddUnconfirmedServerSideHitMarkers（客户端记录未确认的命中位置）
  → OnTargetDataReadyCallback
    → CallServerSetReplicatedTargetData（发送到服务端）
    → 服务端验证 → GE 应用
    → ClientConfirmTargetData RPC（服务端确认命中）
      → 命中标记加入 LastWeaponDamageScreenLocations
      → 准心 Widget 读取并渲染
```
