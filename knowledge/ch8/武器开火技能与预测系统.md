# 武器开火技能与预测系统

## 概述

`ULyraGameplayAbility_RangedWeapon` 是远程武器的开火 GameplayAbility，处理完整的射击流程：瞄准追踪、子弹散布、命中判定、伤害应用，并通过 UE 的 GameplayAbility 预测系统实现客户端即时反馈。

## 开火流程

### ActivateAbility

1. 绑定目标数据回调：`AbilityTargetDataSetDelegate(CurrentSpecHandle, ActivationPredictionKey).AddUObject(this, &OnTargetDataReadyCallback)`
2. 调用 `UpdateFiringTime()` — 更新武器的 TimeLastFired
3. 调用父类 `ActivateAbility`

> 回调绑定的意义：当客户端预测激活时，服务端复制 TargetData 后会通过此委托调用回调。客户端本地产生 TargetData 后也会直接调用回调。

### StartRangedWeaponTargeting

触发开火追踪的入口：

1. 创建 `FScopedPredictionWindow` — 开启预测窗口
2. 调用 `PerformLocalTargeting(OutHits)` — 执行本地的子弹追踪
3. 构造 `FGameplayAbilityTargetDataHandle`：
   - 每个命中结果创建 `FLyraGameplayAbilityTargetData_SingleTargetHit`
   - 包含 `CartridgeID`（唯一标识本次弹药）
4. 如果本地控制的客户端：`WeaponStateComponent->AddUnconfirmedServerSideHitMarkers(TargetData, FoundHits)` — 记录未确认的命中标记
5. 调用 `OnTargetDataReadyCallback(TargetData, FGameplayTag())`

### PerformLocalTargeting

1. 获取瞄准方向：通过 `GetTargetingTransform()` 获取 Camera 朝向 Focus 的变换
2. 计算起始点 `StartTrace` 和终点 `EndAim`（MaxDamageRange 距离）
3. 调用 `TraceBulletsInCartridge(FRangedWeaponFiringInput, OutHits)`

### TraceBulletsInCartridge

**参数：**
- `FRangedWeaponFiringInput` — 包含 StartTrace、EndAim、AimDir、WeaponData

**流程：**

对每发子弹（BulletsPerCartridge）：
1. 计算扩散方向：`VRandConeNormalDistribution(AimDir, SpreadAngleRad, SpreadExponent)`
2. 计算实际终点：`EndTrace = StartTrace + SpreadDir * MaxDamageRange`
3. 调用 `DoSingleBulletTrace(StartTrace, EndTrace, SweepRadius, bIsSimulated, OutHits)`

### DoSingleBulletTrace

**双阶段追踪策略：**

1. **第一阶段 — 射线追踪（Ray Trace）**：
   - 使用 `WeaponTrace()`
   - 调用 `FindFirstPawnHitResult(Hits)` 检查是否命中 Pawn
2. **第二阶段 — 范围追踪（Sweep Trace）**（仅在射线未命中 Pawn 且 SweepRadius > 0 时）：
   - 使用 `SweepRadius` 进行扫描追踪
   - 如果扫描命中 Pawn，使用扫描结果；否则保留射线结果

### FindFirstPawnHitResult

在 HitResults 数组中查找第一个被击中的 Pawn（非自身、非 ignored actor）。

### WeaponTrace

- 使用 `GetWorld()->LineTraceMultiByChannel` 或 `SweepMultiByChannel`
- 追踪通道通过 `DetermineTraceChannel()` 确定
- 忽略 Actor 通过 `AddAdditionalTraceIgnoreActors(TraceParams)` 添加

### OnTargetDataReadyCallback

接收到目标数据后的处理：

1. **服务端**（`HasAuthority(&ActivationInfo)`）：
   - 创建 `FScopedPredictionWindow`（服务端预测窗口）
2. **客户端→服务端同步**（本地控制 && 非权威）：
   - 调用 `CallServerSetReplicatedTargetData()` — RPC 将 TargetData 发送到服务端
3. **服务端确认命中**：
   - 通过 `WeaponStateComponent->ClientConfirmTargetData(UniqueId, bSuccess, HitReplaces)` 确认命中标记
4. **CommitAbility** — 消耗成本（如弹药）
5. **AddSpread** — 增加武器扩散
6. **触发蓝图事件**：`OnRangedWeaponTargetDataReady(TargetData)` — BlueprintImplementableEvent
   - 蓝图中：遍历 TargetData，对每个命中目标施加伤害 GE
7. **清理**：`ConsumeClientReplicatedTargetData()`

## 伤害应用与 GEEC

- 开火技能本身不直接计算伤害，而是通过 `ApplyGameplayEffectToTarget` 将伤害 GE 应用到命中目标
- 伤害计算由 `ULyraDamageExecution`（GEEC）执行：
  - **BaseDamage**：来自 `ULyraCombatSet` 的 BaseDamage 属性
  - **DistanceAttenuation**：从 `ILyraAbilitySourceInterface`（RangedWeaponInstance）获取
  - **PhysicalMaterialAttenuation**：从 `ILyraAbilitySourceInterface`（RangedWeaponInstance）获取

## 成本系统（ULyraAbilityCost_ItemTagStack）

- **CheckCost**：检查 InventoryItemInstance 上是否有足够的 TagStack
- **ApplyCost**：消耗对应的 TagStack
- `bOnlyApplyCostOnHit`：结合 `DetermineIfAbilityHitTarget` lambda，仅在命中目标时消耗

## 失败通知

当 Ability 激活失败时通过 NotifyAbilityFailed 通知：
- `ClientNotifyAbilityFailed` RPC → `HandleAbilityFailed`
- 广播 `TAG_ABILITY_SIMPLE_FAILURE_MESSAGE`（HUD 简单失败提示）
- 广播 `TAG_ABILITY_PLAY_MONTAGE_FAILURE_MESSAGE`（Montage 失败提示）

## 瞄准源类型（ELyraAbilityTargetingSource）

```cpp
enum class ELyraAbilityTargetingSource : uint8
{
    CameraTowardsFocus,    // 从摄像机看向焦点
    PawnForward,           // 从 Pawn 中心朝向 Pawn 方向
    PawnTowardsFocus,      // 从 Pawn 中心看向焦点
    WeaponForward,         // 从武器枪口朝向 Pawn 方向
    WeaponTowardsFocus,    // 从武器枪口看向焦点
    Custom                 // 蓝图自定义
};
```

## GE 应用注意事项

GE 通过 `FGameplayAbilityTargetData::ApplyGameplayEffectSpec` 应用时，会为每个目标复制一份 `EffectContext`（`EffectContext.Duplicate()`），避免伤害累加问题。
