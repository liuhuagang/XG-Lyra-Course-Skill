# 足迹上下文系统

## 概述

ContextEffects（足迹上下文效果）系统是 Lyra 中根据物理表面类型和 GameplayTag 上下文，动态播放脚步声、粒子特效的反馈系统。系统覆盖从动画通知触发 → 接口抽象 → 组件管理 → 资源库匹配 → 子系统生成效果的完整链路。

## 系统架构

```
UAnimNotify_LyraContextEffects (动画通知)
    → ILyraContextEffectsInterface::AnimMotionEffect (接口)
    → ULyraContextEffectComponent::AnimMotionEffect_Implementation (组件实现)
    → ULyraContextEffectsSubsystem::SpawnContextEffects (子系统)
    → ULyraContextEffectsLibrary::GetEffects (资源库匹配)
    → UGameplayStatics::SpawnSoundAttached / UNiagaraFunctionLibrary::SpawnSystemAttached
```

数据资产，存储 `EffectTag + Context(TagContainer) → Sound/NiagaraSystem` 的映射关系。

### 关键结构

```cpp
USTRUCT(BlueprintType)
struct FLyraContextEffects
{
    FGameplayTag EffectTag;                      // 效果标签（如 Footstep.Run）
    FGameplayTagContainer Context;               // 上下文标签（如 SurfaceType.Concrete）
    TArray<FSoftObjectPath> Effects;             // 软引用 SoundBase / NiagaraSystem
};
```

### 匹配逻辑

使用 `MatchesTagExact` + `HasAllExact` 精确匹配，非层级匹配：

```cpp
if (Effect.MatchesTagExact(ActiveContextEffect->EffectTag)
    && Context.HasAllExact(ActiveContextEffect->Context)
    && (ActiveContextEffect->Context.IsEmpty() == Context.IsEmpty()))
{
    Sounds.Append(ActiveContextEffect->Sounds);
    NiagaraSystems.Append(ActiveContextEffect->NiagaraSystems);
}
```

## 组件层：ULyraContextEffectComponent

[LyraContextEffectComponent.h](file:///d:/UPS/GitLab/XGUnrealNote/Courses/Lyra-Deep-Dive/code/Lyra/Source/LyraGame/Feedback/ContextEffects/LyraContextEffectComponent.h) / [LyraContextEffectComponent.cpp](file:///d:/UPS/GitLab/XGUnrealNote/Courses/Lyra-Deep-Dive/code/Lyra/Source/LyraGame/Feedback/ContextEffects/LyraContextEffectComponent.cpp)

实现 `ILyraContextEffectsInterface` 的 ActorComponent，负责：

1. **上下文聚合**：将传入 Contexts + 组件自身 `CurrentContexts` + 物理表面类型三重聚合
2. **物理表面映射**：通过 `bConvertPhysicalSurfaceToContext` 开关，将 `HitResult.PhysMaterial` 的表面类型从 `ULyraContextEffectsSettings` 配置映射为 GameplayTag
3. **生命周期管理**：`BeginPlay` 时加载 `DefaultContextEffectsLibraries` 到 Subsystem，`EndPlay` 时卸载

## 动画通知：UAnimNotify_LyraContextEffects

[AnimNotify_LyraContextEffects.h](file:///d:/UPS/GitLab/XGUnrealNote/Courses/Lyra-Deep-Dive/code/Lyra/Source/LyraGame/Feedback/ContextEffects/AnimNotify_LyraContextEffects.h) / [AnimNotify_LyraContextEffects.cpp](file:///d:/UPS/GitLab/XGUnrealNote/Courses/Lyra-Deep-Dive/code/Lyra/Source/LyraGame/Feedback/ContextEffects/AnimNotify_LyraContextEffects.cpp)

动画通知，在动画播放到特定帧时触发。支持可选的 LineTrace 检测获取 HitResult 和物理材质。

### 执行流程

1. **可选 Trace**：通过 `bPerformTrace` 控制是否执行 LineTrace
2. **查找接口实现**：遍历 Owner 及其 Components，查找实现 `ILyraContextEffectsInterface` 的对象
3. **分发调用**：调用每个实现对象的 `AnimMotionEffect`
4. **编辑器预览**：在编辑器中直接从 Library 获取并生成效果

### 辅助结构体

- `FLyraContextEffectAnimNotifyVFXSettings`：Niagara 缩放
- `FLyraContextEffectAnimNotifyAudioSettings`：音量/音调倍率
- `FLyraContextEffectAnimNotifyTraceSettings`：TraceChannel、偏移、忽略 Actor
- `FLyraContextEffectAnimNotifyPreviewSettings`：编辑器预览配置

## 关键设计要点

1. **接口解耦**：动画通知不直接依赖组件类型，通过接口发现实现对象
2. **精确匹配**：Library 使用 `MatchesTagExact` + `HasAllExact`，确保精确匹配而非层级继承
3. **上下文聚合**：传入 Contexts + 默认 Contexts + 物理表面类型三重聚合
4. **生命周期绑定**：Component BeginPlay/EndPlay 自动注册/注销 Libraries
5. **可配置映射**：物理表面映射通过 `UDeveloperSettings` 在 `DefaultGame.ini` 配置
