# 回放系统

## 概述

Lyra 的回放系统基于 UE5 的 DemoNetDriver 回放框架，通过 `ULyraReplaySubsystem` 管理录制、播放、清理等核心功能，并提供蓝图异步节点查询可用回放列表。回放播放时使用 `ALyraReplayPlayerController` 自动跟随录制者的 Pawn 切换视角。

## ULyraReplaySubsystem

[LyraReplaySubsystem.h](file:///d:/UPS/GitLab/XGUnrealNote/Courses/Lyra-Deep-Dive/code/Lyra/Source/LyraGame/Replays/LyraReplaySubsystem.h) / [LyraReplaySubsystem.cpp](file:///d:/UPS/GitLab/XGUnrealNote/Courses/Lyra-Deep-Dive/code/Lyra/Source/LyraGame/Replays/LyraReplaySubsystem.cpp)

`UGameInstanceSubsystem`，作为 GameInstance 级别的回放管理入口。

### 核心方法

- **DoesPlatformSupportReplays**：通过 `ICommonUIModule::GetSettings().GetPlatformTraits()` 检查 `Platform.Trait.ReplaySupport` GameplayTag，判断当前平台是否支持回放
- **PlayReplay**：通过 `GetGameInstance()->PlayReplay(DemoName)` 播放指定回放
- **RecordClientReplay**：触发 `GetGameInstance()->StartRecordingReplay()` 开始录制，然后从 `ULyraLocalPlayer` 的本地设置中读取保留数量，触发旧回放清理
- **CleanupLocalReplays**：使用 `FNetworkReplayStreaming::Get().GetFactory().CreateReplayStreamer()` 创建流播放器，枚举所有回放流，排序后删除超出保留数量的旧回放
- **SeekInActiveReplay**：委托给 `UDemoNetDriver::GotoTimeInSeconds()`
- **GetReplayLengthInSeconds** / **GetReplayCurrentTime**：委托给 `UDemoNetDriver` 获取回放时长和当前位置

### 数据结构

- **ULyraReplayListEntry**：包装 `FNetworkReplayStreamInfo` 的 UObject，提供 BlueprintPure 访问方法（`GetFriendlyName`、`GetTimestamp`、`GetDuration`、`GetNumViewers`、`GetIsLive`）
- **ULyraReplayList**：回放查询结果集合，包含 `TArray<TObjectPtr<ULyraReplayListEntry>> Results`

## UAsyncAction_QueryReplays

[AsyncAction_QueryReplays.h](file:///d:/UPS/GitLab/XGUnrealNote/Courses/Lyra-Deep-Dive/code/Lyra/Source/LyraGame/Replays/AsyncAction_QueryReplays.h) / [AsyncAction_QueryReplays.cpp](file:///d:/UPS/GitLab/XGUnrealNote/Courses/Lyra-Deep-Dive/code/Lyra/Source/LyraGame/Replays/AsyncAction_QueryReplays.cpp)

Blueprint 异步节点，用于查询可用回放列表：

1. 静态工厂方法 `QueryReplays` 创建实例
2. `Activate()` 创建 `ReplayStreamer`，调用 `EnumerateStreams` 查询
3. `OnEnumerateStreamsComplete` 遍历结果，创建 `ULyraReplayListEntry` 添加到列表，按时间戳降序排序，广播 `QueryComplete` 委托

## ALyraReplayPlayerController

[LyraPlayerController.h](file:///d:/UPS/GitLab/XGUnrealNote/Courses/Lyra-Deep-Dive/code/Lyra/Source/LyraGame/Player/LyraPlayerController.h#L143-L164) / [LyraPlayerController.cpp](file:///d:/UPS/GitLab/XGUnrealNote/Courses/Lyra-Deep-Dive/code/Lyra/Source/LyraGame/Player/LyraPlayerController.cpp#L549-L603)

回放播放时的 PlayerController，自动跟随录制者的 Pawn 切换视角。

### 注册方式

在 [LyraGameMode.cpp](file:///d:/UPS/GitLab/XGUnrealNote/Courses/Lyra-Deep-Dive/code/Lyra/Source/LyraGame/GameModes/LyraGameMode.cpp#L39) 中：

```cpp
ReplaySpectatorPlayerControllerClass = ALyraReplayPlayerController::StaticClass();
```

### 跟随逻辑

- **Tick**：每帧检查 `FollowedPlayerState` 有效性，无效时从 `ALyraGameState` 获取 `RecorderPlayerState` 并重新绑定
- **RecorderPlayerStateUpdated**：保存要跟随的 PlayerState，绑定其 `OnPawnSet` 委托
- **OnPlayerStatePawnSet**：当被跟随的 PlayerState 切换 Pawn 时，调用 `SetViewTarget(NewPlayerPawn)` 切换视角
- **ShouldRecordClientReplay**：返回 false，表示回放 Spectator 不参与录制
