# 炸弹人玩法（俯视角竞技场）

## 概述

TopDownArena 是 Lyra 的一个 Game Feature 插件（`Plugins/GameFeatures/TopDownArena`），实现了一个**炸弹人风格的俯视角竞技场玩法**。该玩法的特点是：C++ 层仅负责基础设施（属性定义、移动、相机、数据容器），游戏逻辑层完全通过 Blueprint 和 GAS 编排。

## C++ 基础设施

### 属性集

[UTopDownArenaAttributeSet](file:///d:/UPS/GitLab/XGUnrealNote/Courses/Lyra-Deep-Dive/code/Lyra/Plugins/GameFeatures/TopDownArena/Source/TopDownArenaRuntime/Private/TopDownArenaAttributeSet.h) 继承 `ULyraAttributeSet`，定义四种属性，全部使用 `DOREPLIFETIME_CONDITION_NOTIFY` + `REPNOTIFY_Always` 同步：

| 属性 | 默认值 | 钳制范围 | 说明 |
|------|--------|----------|------|
| `BombsRemaining` | 1.0 | `[0, BombCapacity]` | 剩余可放置炸弹数 |
| `BombCapacity` | 1.0 | `>= 1` | 最大同时炸弹数 |
| `BombRange` | 2.0 | `>= 1` | 爆炸范围 |
| `MovementSpeed` | 400.0 | `[200, 800]` | 移动速度 |

钳制逻辑在 `PreAttributeBaseChange` / `PreAttributeChange` 中通过 `ClampAttribute` 统一处理。

### 移动组件

[UTopDownArenaMovementComponent](file:///d:/UPS/GitLab/XGUnrealNote/Courses/Lyra-Deep-Dive/code/Lyra/Plugins/GameFeatures/TopDownArena/Source/TopDownArenaRuntime/Private/TopDownArenaMovementComponent.h) 继承 `ULyraCharacterMovementComponent`，重写 `GetMaxSpeed()`：

1. 通过 `UAbilitySystemGlobals::GetAbilitySystemComponentFromActor` 获取 ASC
2. 检测 `TAG_Gameplay_MovementStopped` Tag：命中时返回 0（停止移动）
3. 否则读取 `MovementSpeed` 属性值作为最大速度

**属性驱动移动速度**的典型 GAS 模式。

### 相机模式

[ULyraCameraMode_TopDownArenaCamera](file:///d:/UPS/GitLab/XGUnrealNote/Courses/Lyra-Deep-Dive/code/Lyra/Plugins/GameFeatures/TopDownArena/Source/TopDownArenaRuntime/Public/LyraCameraMode_TopDownArenaCamera.h) 继承 `ULyraCameraMode`（`Abstract, Blueprintable`），固定俯视竞技场模式：

- `ArenaWidth` / `ArenaHeight`：竞技场尺寸
- `DefaultPivotRotation`：默认俯视角旋转
- `BoundsSizeToDistance`（`FRuntimeFloatCurve`）：竞技场大小到相机距离的映射曲线

`UpdateView` 实现：根据包围盒最大边长通过曲线评估相机距离 → 计算 Pivot 位置 → 设置 View。

**不跟随玩家，锁定整个竞技场范围。**

### 拾取物 UI 数据

[UTopDownArenaPickupUIData](file:///d:/UPS/GitLab/XGUnrealNote/Courses/Lyra-Deep-Dive/code/Lyra/Plugins/GameFeatures/TopDownArena/Source/TopDownArenaRuntime/Private/TopDownArenaPickupUIData.h) 继承 `UGameplayEffectUIData`，挂载在 GameplayEffect 上：

- `PickupVFX`（`UNiagaraSystem`）：拾取特效
- `PickupSFX`（`USoundBase`）：拾取音效

## Blueprint 游戏逻辑

### 炸弹机制

炸弹逻辑全部使用 Blueprint 实现，核心资产在 `Game/Bombs/` 目录：

- `GA_DropBomb`（GameplayAbility）：放置炸弹技能
  - 放置时应用 `GE_DecrementBombsRemaining` 减少 `BombsRemaining`
  - 检测 `Ability.ActivateFail.CellAlreadyContainsBomb` Tag 防止同一格放多个炸弹
- `B_Bomb_Base` / `B_Bomb_Standard`：炸弹 Actor 基类和标准炸弹
- 炸弹三状态：**已放置** → **燃烧** → **爆炸**
- 爆炸逻辑：
  1. 四方向递归爆炸，每次波及相邻一格，直至达到最大爆炸半径
  2. 每个方向有 `bool` 数组控制是否继续递归（遇到障碍物停止）
  3. 重叠检测：静态网格体（阻塞）→ 空位（生成火球）→ 其他炸弹（触发连锁）→ 可破坏物（概率生成拾取物）→ 角色（造成伤害）
- 爆炸后应用 `GE_IncrementBombsRemaining` 恢复 `BombsRemaining`
- 通过 `EndOverlap` 检测避免放置炸弹时卡住自己

### 拾取物

拾取物通过 GE 定义，放置在 `Game/Pickups/` 目录：

- `GE_BombRangeUp`：提升爆炸范围
- `GE_BombCountUp`：提升炸弹数量上限
- 拾取时通过 Attribute 同步修改 UI 样式
- 使用 `GameplayCue.TopDownArenaGame.PickupAcquired` 触发拾取特效

### Experience 与 GameMode

使用 Lyra 标准模式，无 C++ GameMode：

- `B_TopDownArenaExperience` / `B_TopDownArena_Multiplayer_Experience` 定义经验和多人支持
- `B_TopDownArena_GameComponent_Base` / `B_TopDownArena_GameComponent_Lives` / `B_TopDownArena_GameComponent_SpawnLocalPlayers` 管理游戏规则
- `DA_TopDownArena_Splode` / `DA_TopDownArena_Multiplayer` 作为 Playlist 数据资产

### 输入资产

定义俯视角炸弹人专用输入映射，支持炸弹放置、移动等操作。

### UI 布局

使用 UIExtensionPoint 系统动态组合 HUD：
- `TopDownArena.ExtensionPoint.AbilityBar`：技能栏
- `TopDownArena.ExtensionPoint.Players`：玩家列表
- `TopDownArena.ExtensionPoint.StatsBar`：状态栏

`OnAddOrRemoveExtension` 回调中根据 Data 类型实例化或移除控件，通过 `ConfigureWidgetForData` 绑定属性数据。

### 关卡生成器

参数化网格生成器，适用于 `L_TopDown_LocalMultiplayer` 关卡，计算网格后进行可破坏物放置。

### 队伍与积分

- 支持 4 队伍（`B_TopDownArena_GameComponent_Base` 管理积分）
- 阶段切换系统管理游戏流程（等待 → 准备 → 进行中 → 结束）
- 人物选择界面

## GameplayTags

| Tag | 用途 |
|-----|------|
| `Ability.ActivateFail.CellAlreadyContainsBomb` | 防止同一格放多个炸弹 |
| `GameplayCue.TopDownArenaGame.PickupAcquired` | 拾取物获取 GC 通知 |
| `GameplayCue.TopDownArenaGame.UserMessage.WaitingForPlayers` | 等待消息 |
| `GameplayCue.TopDownArenaGame.UserMessage.GetReady` | 准备消息 |
| `GameplayCue.TopDownArenaGame.UserMessage.GameOver` | 游戏结束消息 |
| `TopDownArena.ExtensionPoint.*` | UI 扩展点定义 |

## 架构特点

- **C++ 层极简**：仅 5 个类、约 280 行代码，负责数值框架和核心机制接口
- **BP 层编排**：炸弹、拾取物、GameMode、Experience 全部使用 Blueprint
- **GAS 驱动**：通过 GameplayEffect 操控 AttributeSet，GameplayCue 处理视觉反馈
- **Game Feature 独立封装**：完整的自包含玩法插件
