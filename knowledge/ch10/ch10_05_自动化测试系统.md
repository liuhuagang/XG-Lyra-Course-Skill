# 自动化测试系统

## 概述

Lyra 的自动化测试体系包括两套框架：基于 CQTest 的功能测试（位于 ShooterTests 插件）和基于 Gauntlet 的启动测试。CQTest 提供结构化的功能测试框架，支持地图测试、动画测试、网络测试、输入模拟和设备属性测试。Gauntlet 提供自动化部署和启动检测。

## CQTest 功能测试框架

CQTest 是 UE5 内置的现代化测试框架，Lyra 在 `ShooterTests` 插件中集成。

### 测试基础模式

```cpp
TEST_CLASS_WITH_FLAGS(TestClassName, "ProjectName.TestCategory", EAutomationTestFlags::ApplicationContextMask | EAutomationTestFlags::ProductFilter)
BEFORE_EACH
{
    // 使用 FMapTestSpawner 在测试地图中生成角色
    MapTestSpawner = MakeUnique<FMapTestSpawner>(TEXT("/Game/Maps/TestMap"));
    MapTestSpawner->BeginSpawning();
}
AFTER_EACH { }

TEST_METHOD(TestMethodName)
{
    // 执行伤害测试：通过 SetByCaller Magnitude 设置伤害值
    DoDamageToPlayer(TargetPlayer, DamageAmount);
    ASSERT_THAT(Health, NumericEqual(ExpectedHealth));
}
```

## Gauntlet 自动化框架

Gauntlet 是 UE5 的自动化测试框架，Lyra 通过 `ULyraTestControllerBootTest` 实现启动检测。

### 执行流程

```
RunUAT
    → RunUnreal with -test=LyraTestControllerBootTest
    → 游戏启动，Gauntlet 控制器等待 TestDelay（20 秒）
    → IsBootProcessComplete() 返回 true → 测试通过
```

### ULyraTestControllerBootTest

[LyraTestControllerBootTest.h](file:///d:/UPS/GitLab/XGUnrealNote/Courses/Lyra-Deep-Dive/code/Lyra/Source/LyraGame/Tests/LyraTestControllerBootTest.h) / [LyraTestControllerBootTest.cpp](file:///d:/UPS/GitLab/XGUnrealNote/Courses/Lyra-Deep-Dive/code/Lyra/Source/LyraGame/Tests/LyraTestControllerBootTest.cpp)

继承自 `UGauntletTestControllerBootTest`，重写 `IsBootProcessComplete()` 方法。当前实现为占位状态，通过 20 秒硬编码延迟判断启动完成，实际的有用测试逻辑尚未实现（TODO 注释标记，理想实现应检测 `ULyraGameInstance` 是否进入欢迎屏幕或主菜单）。

```cpp
UCLASS()
class ULyraTestControllerBootTest : public UGauntletTestControllerBootTest
{
    const double TestDelay = 20.0f;

    virtual bool IsBootProcessComplete() const override
    {
        double TimeSinceStart = FPlatformTime::Seconds() - StartTime;
        return TimeSinceStart >= TestDelay;
    }
};
```
