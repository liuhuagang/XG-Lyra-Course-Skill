# 作弊调试系统

## 概述

Lyra 的作弊调试系统基于 UE5 的 `UCheatManager` 框架，通过 `ULyraCheatManager` 提供核心作弊命令，并通过 `UCheatManagerExtension` 机制将不同子系统的作弊功能分散到独立的扩展类中。此外，还提供基于 HTTP Server 的外部 RPC 注册组件，支持远程调用游戏内命令。

### 核心功能

- **调试相机切换**：`CycleDebugCameras` 按 Normal → Debug → Fixed → Normal 循环，Fixed Camera 模式通过 `SetCameraMode` 切换 `NAME_Fixed` / `NAME_Default`
- **上帝模式 / 无限血量**：通过 `ULyraAbilitySystemComponent` 的动态 GameplayTag 机制实现，使用 `AddDynamicTagGameplayEffect` 应用 `Cheat_GodMode` / `Cheat_UnlimitedHealth` 标签
- **伤害 / 治疗**：通过 `SetByCaller` 的 GameplayEffect，标签为 `LyraGameplayTags::SetByCaller_Damage` / `SetByCaller_Heal`
- **自毁**：通过 `ULyraHealthComponent::DamageSelfDestruct()`，检查 Pawn 是否已达到 `InitState_GameplayReady` 状态
- **ASC 调试**：提供 `CycleAbilitySystemDebug`、`CancelActivatedAbilities`、`AddTagToSelf`、`RemoveTagFromSelf` 等命令
- **Editor 自动执行**：`InitCheatManager` 中从 `ULyraDeveloperSettings` 读取 `CheatsToRun` 数组，自动执行 `OnCheatManagerCreated` 阶段的 cheat 命令

## CheatManagerExtension 扩展机制

Lyra 利用 `UCheatManager::RegisterForOnCheatManagerCreated` 静态委托在 CDO 构建时自动注册扩展。每个扩展类在其构造函数中调用此委托，当 CheatManager 创建时自动将扩展添加到其中。

### 注册模式

```cpp
ULyraBotCheats::ULyraBotCheats()
{
    if (HasAnyFlags(RF_ClassDefaultObject))
    {
        UCheatManager::RegisterForOnCheatManagerCreated(FOnCheatManagerCreated::FDelegate::CreateLambda(
            [](UCheatManager* CheatManager)
            {
                CheatManager->AddCheatManagerExtension(NewObject<ThisClass>(CheatManager));
            }));
    }
}
```

## ULyraGameplayRpcRegistrationComponent 外部 RPC

[LyraGameplayRpcRegistrationComponent.h](file:///d:/UPS/GitLab/XGUnrealNote/Courses/Lyra-Deep-Dive/code/Lyra/Source/LyraGame/Tests/LyraGameplayRpcRegistrationComponent.h) / [LyraGameplayRpcRegistrationComponent.cpp](file:///d:/UPS/GitLab/XGUnrealNote/Courses/Lyra-Deep-Dive/code/Lyra/Source/LyraGame/Tests/LyraGameplayRpcRegistrationComponent.cpp)

基于 `UExternalRpcRegistrationComponent` 的 HTTP Server 组件，支持外部工具远程调用游戏内命令。

### 核心功能

- **HttpExecuteCheatCommand**：`POST /core/cheatcommand`，解析 JSON Body 中的 `"command"` 字段，通过 `ALyraPlayerController->ConsoleCommand()` 执行
- **HttpGetPlayerVitalsCommand**：`GET /player/status`，返回玩家血量和背包信息
- **HttpFireOnceCommand**：`POST /player/status`，示例 RPC（TODO 状态）

### 注册分层

- `RegisterAlwaysOnHttpCallbacks`：核心命令始终可用
- `RegisterFrontendHttpCallbacks`：前端大厅阶段（TODO）
- `RegisterInMatchHttpCallbacks`：比赛中

### 单例模式

通过静态 `ObjectInstance` 指针和 `GetInstance()` 方法实现单例，`AddToRoot()` 防止 GC。
