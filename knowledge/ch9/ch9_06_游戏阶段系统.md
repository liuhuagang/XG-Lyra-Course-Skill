# 游戏阶段系统

## 架构概览

游戏阶段系统使用 GAS 能力驱动游戏流程的阶段切换，支持阶段嵌套（父阶段存活、子阶段切换）、观察者模式和阶段互斥。

- 每个阶段能力绑定一个 GameplayTag
- Tag 的层级结构决定阶段的父子关系

### Tag 匹配规则

```cpp
enum class EPhaseTagMatchType : uint8
{
    ExactMatch,     // 精确匹配，如 "A.B" 只匹配 "A.B"
    PartialMatch    // 部分匹配，如 "A.B" 匹配 "A.B" 和 "A.B.C"
};
```

## 阶段子系统

[ULyraGamePhaseSubsystem](file:///d:/UPS/GitLab/XGUnrealNote/Courses/Lyra-Deep-Dive/code/Lyra/Source/LyraGame/)

```cpp
class ULyraGamePhaseSubsystem : public UWorldSubsystem
{
    // 激活阶段
    void StartPhase(TSubclassOf<ULyraGamePhaseAbility> PhaseAbility,
                    FLyraGamePhaseDelegate PhaseEndedCallback = FLyraGamePhaseDelegate());

    // 观察阶段开始
    void WhenPhaseStartsOrIsActive(FGameplayTag PhaseTag, EPhaseTagMatchType MatchType,
                                   const FLyraGamePhaseTagDelegate& WhenPhaseActive);

    // 观察阶段结束
    void WhenPhaseEnds(FGameplayTag PhaseTag, EPhaseTagMatchType MatchType,
                       const FLyraGamePhaseTagDelegate& WhenPhaseEnd);

    // 阶段是否激活
    bool IsPhaseActive(const FGameplayTag& PhaseTag) const;

private:
    TMap<FGameplayAbilitySpecHandle, FLyraGamePhaseEntry> ActivePhaseMap;
    TArray<FPhaseObserver> PhaseStartObservers;
    TArray<FPhaseObserver> PhaseEndObservers;
};
```

### 激活流程

1. **蓝图启动**：`K2_StartPhase` 通过弱对象指针包装回调后调用 `StartPhase`
2. **转发到 GA**：`StartPhase` → 获取 GameState 的 ASC → `GiveAbilityAndActivateOnce`
3. **转发到子系统**：GA 的 `ActivateAbility` → `PhaseSubsystem->OnBeginPhase`
4. **互斥处理**：遍历 `ActivePhaseMap`，使用 Tag 层级比较

### 阶段互斥逻辑

```
示例：当前激活 Game.A
- 新阶段 Game.A.1 → Game.A 保持激活（父阶段），Game.A.1 激活（子阶段）
- 新阶段 Game.B   → Game.A 被取消（旁支兄弟）
- 新阶段 Game.A.2 → Game.A 保持激活，Game.A.1 被取消（同级替换）
```

核心比较：
```cpp
if (!IncomingPhaseTag.MatchesTag(ActivePhaseTag))
{
    // 不是子阶段关系 → 取消当前阶段
    GameState_ASC->CancelAbilitiesByFunc(...);
}
```

### 结束流程

1. GA 的 `EndAbility` → `PhaseSubsystem->OnEndPhase`
2. 从 `ActivePhaseMap` 移除
3. 执行阶段结束回调
4. 通知 `PhaseEndObservers`

### 观察者机制

```cpp
struct FPhaseObserver
{
    FGameplayTag PhaseTag;
    EPhaseTagMatchType MatchType;
    FLyraGamePhaseTagDelegate PhaseCallback;
};
```

- `WhenPhaseStartsOrIsActive`：如果阶段已激活立即调用，否则等待激活
- `WhenPhaseEnds`：等到阶段结束后调用
- 注意：`WhenPhaseStartsOrIsActive` 的初始检查使用部分匹配，存在设计歧义

## 游戏阶段示例

### 暖场阶段

- 检查是否跳过全流程（`ShouldSkipDirectlyToGameplay`）
- 施加全局无敌效果 `GE_PregameLobby`
- 播放等待 GameplayCue（循环音效）
- 发送 3 秒倒计时
- 结束后收回无敌 → 进入战斗

### 战斗阶段

- 根据玩家人数设计游戏时长和目标
- 激活人物的 Reset 能力
- 清理场景杂项（手榴弹、武器生成器等）

### 计分阶段

- 时间到/比分到/强制结束触发
- 全局无敌，禁止释放技能
- 延迟几秒后调用 `PlayNextGame` → ServerTravel 开始下一场
