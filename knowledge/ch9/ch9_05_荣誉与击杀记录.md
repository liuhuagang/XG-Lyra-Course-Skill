# 荣誉系统与击杀记录

## 架构概览

荣誉系统处理游戏中的击杀/助攻/连杀/连胜等事件的识别、传递和 UI 播报。核心基于 `UGameplayMessageSubsystem` 的 VerbMessage 系统，通过 GameplayTag 驱动消息路由。

## 荣誉系统

### 荣誉定义

[Accolade 相关文件]

荣誉定义存储在 DataRegistry 中，使用 `FLyraAccoladeDefinitionRow` 表格行：

```cpp
USTRUCT(BlueprintType)
struct FLyraAccoladeDefinitionRow : public FTableRowBase
{
    // 定义荣誉显示的文本、图标、Tag 等
};
```

通过 `UGameFeatureAction_DataRegistry` / `UGameFeatureAction_DataRegistrySource` 将荣誉数据注册到 DataRegistry。

### 荣誉播报 Widget

[ULyraAccoladeHostWidget]

- 继承 `FAsyncMixin`，使用异步加载资源
- 接收 `FLyraNotificationMessage` 消息
- 支持荣誉排队（多条连续荣誉按顺序显示）
- 互斥逻辑：前一条未完成时新荣誉等待

```cpp
struct FLyraNotificationMessage
{
    FGameplayTag TargetChannel;
    TObjectPtr<APlayerState> TargetPlayer;     // 指定目标玩家，空则所有玩家可见
    FText PayloadMessage;
    FGameplayTag PayloadTag;
    TObjectPtr<UObject> PayloadObject;
};
```

## 击杀记录系统

### VerbMessage 消息系统

使用 `UGameplayMessageSubsystem` 的 `BroadcastMessage` 传递结构化消息。

```cpp
USTRUCT(BlueprintType)
struct FLyraVerbMessage
{
    FGameplayTag Verb;
    TObjectPtr<UObject> Instigator;
    FGameplayTagContainer InstigatorTags;
    TObjectPtr<UObject> Target;
    FGameplayTagContainer TargetTags;
    FGameplayTagContainer ContextTags;
    double Magnitude;
};
```

### 标准消息流

```
死亡 → Lyra.Elimination.Message  → AssistProcessor → Lyra.Assist.Message
                                  → ElimChainProcessor → Lyra.ShooterGame.Accolade.EliminationChain.Nx
                                  → ElimStreakProcessor → Lyra.ShooterGame.Accolade.EliminationStreak.N
                                  → EliminationFeedRelay → Lyra.AddNotification.KillFeed
```

### 歼灭消息触发

在 `ULyraHealthComponent::HandleOutOfHealth` 中：

```cpp
FLyraVerbMessage Message;
Message.Verb = TAG_Lyra_Elimination_Message;
Message.Instigator = DamageInstigator;
Message.Target = ULyraVerbMessageHelpers::GetPlayerStateFromObject(AbilitySystemComponent->GetAvatarActor());
Message.InstigatorTags = *DamageEffectSpec->CapturedSourceTags.GetAggregatedTags();
Message.TargetTags = *DamageEffectSpec->CapturedTargetTags.GetAggregatedTags();

UGameplayMessageSubsystem& MessageSystem = UGameplayMessageSubsystem::Get(GetWorld());
MessageSystem.BroadcastMessage(Message.Verb, Message);
```

### 助攻处理器 UAssistProcessor

[UAssistProcessor]

```cpp
class UAssistProcessor : public UGameplayMessageProcessor
{
    TMap<TObjectPtr<APlayerState>, FPlayerAssistDamageTracking> DamageHistory;

    void OnDamageMessage(FGameplayTag Channel, const FLyraVerbMessage& Payload);
    void OnEliminationMessage(FGameplayTag Channel, const FLyraVerbMessage& Payload);
};
```

- 监听 `Lyra.Damage.Message` 积累伤害记录
- 监听 `Lyra.Elimination.Message` 时为所有造成过伤害（非击杀者）的玩家广播助攻消息
- 击杀后清除该目标的伤害记录

### 连杀处理器 UElimChainProcessor

[UElimChainProcessor]

```cpp
class UElimChainProcessor : public UGameplayMessageProcessor
{
    float ChainTimeLimit = 4.5f;                      // 时间窗口
    TMap<int32, FGameplayTag> ElimChainTags;           // 连杀数量 → Tag
    TMap<TObjectPtr<APlayerState>, FPlayerElimChainInfo> PlayerChainHistory;
};
```

- 监听 `Lyra.Elimination.Message`
- 统计连续击杀数（时间窗口内有效）
- 超过时间窗口则重置计数
- 达到配置阈值时广播对应 `ElimChainTags`

### 连胜处理器 UElimStreakProcessor

[UElimStreakProcessor]

```cpp
class UElimStreakProcessor : public UGameplayMessageProcessor
{
    TMap<int32, FGameplayTag> ElimStreakTags;           // 连胜数量 → Tag

```

- 监听 `Lyra.Elimination.Message`
- 统计未被击杀前的连续击杀数
- 目标被击杀时结束其连胜记录

### 客户端播报

```cpp
void ALyraPlayerState::ClientBroadcastMessage_Implementation(const FLyraVerbMessage Message)
{
    if (GetNetMode() == NM_Client)
    {
        UGameplayMessageSubsystem::Get(this).BroadcastMessage(Message.Verb, Message);
    }
}
```

- 使用 PlayerState 的 Client RPC 将消息传递到对应客户端
- 客户端收到后在本地 MessageSubsystem 广播

## 击杀通知 UI

### 击杀 Feed

- 使用 ListView 显示击杀条目
- 服务器和客户端都有 EliminationFeedRelay，但不需要网络同步
- 每条击杀构建信息 Object，设置样式（武器图标、队伍颜色等）
- 支持定时隐退：事件分发器绑定在超时事件上 → 自动移除

## 得分处理

团队歼灭比分组件：

- 监听 `Lyra.Elimination.Message` 和 `Lyra.Assist.Message`
- 修改的比分 Tag：
  - TeamInfo: `ShooterGame.Score.Eliminations`
  - PlayerState: `ShooterGame.Score.Eliminations`
  - PlayerState: `ShooterGame.Score.Deaths`
  - PlayerState: `ShooterGame.Score.Assists`
- 比分通过属性同步下发
