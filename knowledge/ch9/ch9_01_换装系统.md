# 换装系统

## 架构概览

换装系统控制角色的外观组件（头发、装备、装饰等）的添加与移除，支持运行时动态更换和网络复制。

核心文件：[Cosmetics/](file:///d:/UPS/GitLab/XGUnrealNote/Courses/Lyra-Deep-Dive/code/Lyra/Source/LyraGame/Cosmetics/)

## 角色部件类型

### 动画层选择

动画层决定角色的上身/下身/全身上下使用哪套动画蓝图覆盖。

```cpp
USTRUCT(BlueprintType)
struct FLyraAnimLayerSelectionEntry
{
    TSubclassOf<UAnimInstance> Layer;
    FGameplayTagContainer RequiredTags;
};
```

- 支持按 GameplayTag 条件匹配选择动画层
- 不同身体部位可指定不同动画层（如：上身持枪、下身行走）

### 身体风格选择

身体风格决定角色使用哪套骨骼网格体。

```cpp
USTRUCT(BlueprintType)
struct FLyraAnimBodyStyleSelectionEntry
{
    TObjectPtr<USkeletalMesh> Mesh = nullptr;
    FGameplayTagContainer RequiredTags;
};
```

- 按 Tag 条件匹配选择网格体
- 可与动画层组合使用

### 角色部件定义

```cpp
USTRUCT(BlueprintType)
struct FLyraCharacterPart
{
    TSubclassOf<AActor> PartClass;  // 部件类
    FName SocketName;               // 挂载的骨骼插槽
};
```

- 每个部件是一个独立 Actor，挂载到指定插槽上
- 支持运行时添加/移除

## FastArray 复制

部件列表使用 FastArray 序列化实现高效网络复制：

```cpp
USTRUCT()
struct FLyraCharacterPartEntry : public FFastArraySerializerItem
{
    FLyraCharacterPart Part;
    TWeakObjectPtr<AActor> SpawnedActor;  // 生成的部件 Actor
};
```

- 仅同步增/删/改，不传输整个数组
- 客户端 `OnRep` 时按需生成或销毁部件 Actor

## 双组件架构

换装系统由 Controller 和 Pawn 两个组件协作完成。

### Controller 端

[LyraControllerComponent_CharacterParts.h](file:///d:/UPS/GitLab/XGUnrealNote/Courses/Lyra-Deep-Dive/code/Lyra/Source/LyraGame/Cosmetics/LyraControllerComponent_CharacterParts.h)

- 管理角色部件的添加和移除（权限服务器）
- 通过 `AddCharacterPart` 添加部件，`RemoveCharacterPart` 移除
- 部件列表通过 `OnRep_PartList` 同步到客户端
- 支持调试作弊功能：`AddCheatPart`、`ClearCheatParts`

```cpp
UCLASS()
class ULyraControllerComponent_CharacterParts : public UControllerComponent
{
    void AddCharacterPart(const FLyraCharacterPart& Part);
    void RemoveCharacterPart(FLyraCharacterPart Part, bool bRemoveAll = false);

    UPROPERTY(ReplicatedUsing=OnRep_PartList)
    FLyraCharacterPartList PartList;
};
```

### Pawn 端

[LyraPawnComponent_CharacterParts.h](file:///d:/UPS/GitLab/XGUnrealNote/Courses/Lyra-Deep-Dive/code/Lyra/Source/LyraGame/Cosmetics/LyraPawnComponent_CharacterParts.h)

- 负责生成和销毁部件 Actor
- 处理部件挂载插槽和同步动画标签

```cpp
UCLASS()
class ULyraPawnComponent_CharacterParts : public UPawnComponent
{
    void AddCharacterPart(const FLyraCharacterPart& Part, int32& PartHandle);
    TArray<FLyraAppliedCharacterPart> AppliedParts;
};
```

## 工作流程

1. `AddCharacterPart` 在 Controller 组件上调用
2. 服务器将部件添加到 `PartList` FastArray
3. `OnRep_PartList` 触发客户端同步
4. Pawn 组件收到后通过 `SpawnActor` 延迟生成部件 Actor
5. 生成的 Actor 挂载到指定 Socket
6. 应用换装动画标签，影响动画层选择
