# 游戏设置界面与注册器

## 概述

游戏设置界面（`ULyraSettingScreen`）和游戏设置注册器（`ULyraGameSettingRegistry`）构成了 Lyra 设置系统的入口层。设置界面负责展示和交互，注册器负责管理所有设置项的注册、过滤和变更追踪。

## 设置屏幕架构

```
ULyraSettingScreen
  ├── TopSettingsTabs (ULyraTabListWidgetBase) — Tab 切换
  ├── Settings_Panel (UGameSettingPanel) — 设置面板
  └── 底部输入栏 — 返回/应用/取消
```

### 输入绑定

在 `NativeOnInitialized` 中注册三个核心输入绑定：

| 绑定 | 输入动作 | 说明 |
|------|---------|------|
| BackHandle | BackInputActionData | 返回（有修改时先确认） |
| ApplyHandle | ApplyInputActionData | 应用修改 |
| CancelChangesHandle | CancelChangesInputActionData | 取消修改 |

### 脏状态管理

`OnSettingsDirtyStateChanged` 根据是否有未保存的修改动态添加/移除操作按钮：

```
bSettingsDirty = true → AddActionBinding(ApplyHandle) + AddActionBinding(CancelChangesHandle)
bSettingsDirty = false → RemoveActionBinding(ApplyHandle) + RemoveActionBinding(CancelChangesHandle)
```

## 注册器创建

```
ULyraSettingScreen::CreateRegistry()
  └── NewRegistry = NewObject<ULyraGameSettingRegistry>()
        └── NewRegistry->Initialize(LocalPlayer)

UGameSettingScreen::GetOrCreateRegistry()
  └── CreateRegistry() [虚函数，由子类实现]
        ├── OnSettingChangedEvent 绑定
        ├── Settings_Panel->SetRegistry(NewRegistry)
        └── 缓存到 Registry 成员变量
```

## Tab-内容联动流程

```
1. Blueprint: Construct → RegisterTopLevelTab → RegisterDynamicTab(FLyraTabDescriptor)
2. RegisterDynamicTab → PendingTabLabelInfoMap.Add → RegisterTab(...)
3. TabButtonGroup 自动选中首个按钮（bSelectionRequired = true）
4. 点击 → SetSelectedInternal(true) → NativeOnSelected → 广播
5. HandleTabButtonSelected → 切换内容 + 广播 OnTabSelected
6. Blueprint OnTabSelected → NavigateToSetting(SettingDevName)
     └── FindSettingByDevNameChecked
           └── FilterState.AddSettingToRootList(Setting)
                 └── Settings_Panel->SetFilterState(FilterState)
```

## FGameSettingRegistryChangeTracker

- **WatchRegistry**：监听注册器中的设置变更
- **ApplyChanges**：应用所有脏设置
- **RestoreToInitial**：恢复初始状态
- **ClearDirtyState**：清除脏状态
- **DirtySettings**：脏设置映射表（TMap<FObjectKey, TWeakObjectPtr<UGameSetting>>）
- **bRestoringSettings**：恢复中标志位，防止递归
