# 设置面板、列表与细节视图

## 概述

设置面板的显示层由三个核心组件构成：`UGameSettingPanel`（面板容器）、`UGameSettingListView`（设置列表）和 `UGameSettingDetailView`（设置细节视图）。三者协作实现对设置项的浏览、选择和编辑。

## GameSettingPanel 职责

### 面板管理

1. **容纳设置列表**：持有 `ListView_Settings` 显示设置项
2. **容纳细节面板**：持有 `Details_Settings` 显示当前选中设置的详情
3. **事件中转**：中转注册器的三种事件（导航、命名操作、编辑条件变化）

### 过滤器

`SetFilterState(const FGameSettingFilterState& InFilterState, bool bClearNavigationStack)`：
- 设置面板的过滤器状态
- 可选的导航栈管理
- 自动触发列表刷新

### 刷新逻辑

```
RefreshSettingsList():
  ├── 防重入（检查 RefreshHandle）
  ├── 使用 FTSTicker 延后一帧执行
  │     ├── Registry->GetSettingsForFilter(FilterState, VisibleSettings)
  │     ├── ListView_Settings->SetListItems(VisibleSettings)
  │     ├── NavigateToIndex(IndexToSelect) + SetSelectedIndex(IndexToSelect)
  │     └── 刷新所有可见设置的编辑状态
  └── 清理 RefreshHandle
```

### 悬停/选中联动

```
NativeOnInitialized:
  └── 绑定 ListView 事件
        ├── OnItemIsHoveredChanged → HandleSettingItemHoveredChanged → FillSettingDetails
        └── OnItemSelectionChanged → HandleSettingItemSelectionChanged → FillSettingDetails
        └── 转发到 Details_Settings->FillSettingDetails
```

### 注册器事件

| 事件 | 处理函数 | 说明 |
|------|---------|------|
| OnSettingEditConditionChangedEvent | HandleSettingEditConditionsChanged | 编辑条件变化时判断可见性变化，必要时刷新列表 |
| OnSettingNamedActionEvent | HandleSettingNamedAction | 命名操作事件，广播到蓝图 |
| OnExecuteNavigationEvent | HandleSettingNavigation | 导航事件，创建新过滤器推入导航栈 |

### 导航栈

```
HandleSettingNavigation:
  ├── FilterNavigationStack.Push(FilterState) — 保存当前状态
  ├── 创建新过滤器: NewPageFilterState.AddSettingToRootList(Setting)
  └── SetFilterState(NewPageFilterState, false) — 不清除导航栈

PopNavigationStack():
  └── 从导航栈恢复上一个 FilterState
```

## GameSettingListView 职责

### 控件类型映射

通过 `UGameSettingVisualData` 将游戏设置映射到具体的控件蓝图类：

```
OnGenerateEntryWidgetInternal:
  ├── VisualData->GetEntryForSetting(SettingItem)
  │     ├── 自定义逻辑（虚函数 GetCustomEntryForSetting）
  │     ├── 按名称查找（EntryWidgetForName）
  │     └── 按类继承链查找（EntryWidgetForClass）
  ├── GenerateTypedEntry<UGameSettingListEntryBase>
  ├── 名称重写
  └── EntryWidget.SetSetting(SettingItem)
```

### UGameSettingVisualData 查找优先级

1. `GetCustomEntryForSetting`（虚函数，子类自定义）
2. `EntryWidgetForName.FindRef(DevName)`（按名称）
3. `EntryWidgetForClass.FindRef(SettingClass)`（按类继承链，从子类向父类遍历）

## GameSettingDetailView 职责

### 细节面板填充

```
FillSettingDetails(UGameSetting* InSetting):
  ├── 防重复请求（检查 CurrentSetting）
  ├── 设置名称、描述文本
  ├── 动态细节文本（可见性控制）
  ├── 警告文本（带 <Icon.Warning> 格式化）
  ├── 禁用原因文本（遍历 EditableState.GetDisabledReasons）
  └── 创建拓展子控件
        ├── 通过 Box_DetailsExtension 容纳
        ├── 使用 ExtensionWidgetPool 管理 Widget 池
        └── 支持异步加载（通过 UAssetManager::RequestAsyncLoad）
```

### 绑定控件

| 控件 | 类型 | 说明 |
|------|------|------|
| Text_SettingName | UCommonTextBlock | 设置名称 |
| RichText_Description | UCommonRichTextBlock | 设置描述 |
| RichText_DynamicDetails | UCommonRichTextBlock | 动态细节 |
| RichText_WarningDetails | UCommonRichTextBlock | 警告 |
| RichText_DisabledDetails | UCommonRichTextBlock | 禁用原因 |
| Box_DetailsExtension | UVerticalBox | 拓展子控件容器 |
