# 对话框系统

## 概述

Lyra 的对话框系统基于 CommonGame 插件构建，通过消息接收-显示管线（ReceiveMessage → ShouldDisplayMessage → DisplayMessage）管理模态对话框的显示和生命周期。

## 对话框生命周期

```
ReceiveMessage
  → ShouldDisplayMessage (判断是否应该显示)
    → DisplayMessage (创建/显示对话框实例)
```

### 消息接收

对话框通过 Messaging 系统接收消息，消息定义在 `ULyraDialogReceiverDefinition` 类型的 DataAsset 中。

### 显示决策

`ShouldDisplayMessage` 决定是否显示对话框，考虑因素包括：
- 是否有相同优先级的对话框正在显示
- 当前焦点状态
- 对话框优先级

### 对话框显示

`DisplayMessage` 负责：
- 从 Widget Pool 获取或创建对话框实例
- 设置对话框内容
- 管理焦点路径

## 对话框类型

- **确认对话框**（ULyraConfirmationScreen）：确认/取消操作
- **加载对话框**（ULyraLoadingScreen）：加载等待界面

## Widget Pooling

对话框使用 `FUserWidgetPool` 管理动态创建的 Widget 实例，避免频繁创建和销毁。对话框关闭后 Widget 归还到池中，下次显示时复用。
