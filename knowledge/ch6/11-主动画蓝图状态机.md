# 主动画蓝图状态机

## 概述

主动画蓝图的状态机（AnimGraph）是 Lyra Locomotion 系统的核心调度结构，管理角色走、跑、跳、坠落、停止、回转等基本运动状态的切换。对应课程编号 075，具体编辑界面截图见 [原版 .docx 讲义](file:///d:/UPS/GitLab/XGUnrealNote/Courses/Lyra-Deep-Dive/docs/UE5_Lyra学习指南_075_构建主动画蓝图状态机.docx)。

## 状态机架构

```
AnimGraph (主动画蓝图)
  └── Locomotion (监听动画层改变)
        ├── Idle — 空闲
        ├── Start — 起步
        ├── Cycle — 循环（行走/跑步）
        ├── Stop — 停止
        ├── Pivot — 回转
        └── Jump (跳跃)
              ├── JumpStart
              ├── JumpApex
              ├── FallLoop
              └── FallLand
```

## 各状态详解

### Idle（空闲）

- `UpdateIdleState`：
  - 如果正在混出（blend out），将转身曲线值设为 0
  - 如果没有混出，累积转身值，并减少动画已消耗的转身值

### Start（起步）

- `SetUpStartState` — 选择起步动画，初始化参数
- `UpdateStartState` — 推进动画播放

**过渡条件（Start → Cycle）：**
- 自动过渡
- 状态速度过渡
- 动画层变化过渡
- 转身角度过渡

### Cycle（循环）

- 走/跑/跳循环动画
- 由动画层蓝图提供具体的姿态数据

### Stop（停止）

- `UpdateStopState` — 更新停止动画播放

**过渡条件：**
- Cycle → Stop：状态速度判定
- Stop → Start：加速度判定
- Stop → Idle：与上述逻辑大同小异

### Pivot（回转）

- `SetUpPivotState` — 建立回转动画，初始化参数
- `UpdatePivotState` — 回转动画时间由动画层设置

**过渡条件（Pivot → Cycle）：**
- 动画层改变
- 移动状态改变
- 动画通知

**触发回转：** `PivotSources → Pivot`

### Jump（跳跃）

- **JumpStart** — 起跳
- **JumpApex** — 跳跃顶点
- **FallLoop** — 坠落循环
- **FallLand** — 落地

> 判定条件相对简单，注意各状态的别名映射即可

## 状态机与动画层的关系

状态机负责状态调度，但部分逻辑在动画层蓝图中设置回调。主动画蓝图通过**链接动画图层**与动画层交互：

```
状态机调度 (主动画蓝图)
  ↓ 状态变更通知
动画层响应 (动画层蓝图)
  ↓ 姿态数据回传
状态机应用 (主动画蓝图)
```
