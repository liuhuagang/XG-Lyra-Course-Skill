# 角色类层级

## 类层级

```
AModularPawn (Engine)
  └── ALyraPawn : ILyraTeamAgentInterface  — 基础 Pawn，项目中未使用

AModularCharacter (Engine)
  └── ALyraCharacter : IAbilitySystemInterface, IGameplayCueInterface,
                        IGameplayTagAssetInterface, ILyraTeamAgentInterface  — 主角色类
        └── ALyraCharacterWithAbilities  — 自包含 ASC，项目中未使用
```

- `ALyraPawn` 继承了 `AModularPawn`（GameFrameworkComponentManager 集成）和 `ILyraTeamAgentInterface`，实现了基础的团队 ID 复制和 PossessedBy/UnPossessed 时的团队传播
- 项目实际使用的是 `ALyraCharacter`，`ALyraPawn` 和 `ALyraCharacterWithAbilities` 可作为参考但未被使用

## ALyraCharacter 接口实现

### IAbilitySystemInterface

```cpp
UAbilitySystemComponent* ALyraCharacter::GetAbilitySystemComponent() const
{
    // 委托给 PawnExtensionComponent
    return PawnExtComponent ? PawnExtComponent->GetLyraAbilitySystemComponent() : nullptr;
}
```

- `ALyraCharacter` 中不持有 ASC，通过 PawnExtensionComponent 间接获取

### IGameplayTagAssetInterface

四个方法全部委托给 ASC：

```cpp
void GetOwnedGameplayTags(FGameplayTagContainer& TagContainer) const { ASC->GetOwnedGameplayTags(TagContainer); }
bool HasMatchingGameplayTag(FGameplayTag Tag) const { return ASC->HasMatchingGameplayTag(Tag); }
bool HasAllMatchingGameplayTags(const FGameplayTagContainer& TagContainer) const { return ASC->HasAllMatchingGameplayTags(TagContainer); }
bool HasAnyMatchingGameplayTags(const FGameplayTagContainer& TagContainer) const { return ASC->HasAnyMatchingGameplayTags(TagContainer); }
```

### ILyraTeamAgentInterface（队伍系统）

- `ALyraCharacter` 复制 `MyTeamID`（`COND_OwnerOnly`）
- `PossessedBy`：从 Controller 获取 Team ID
- `UnPossessed`：调用 `DetermineNewTeamAfterPossessionEnds` 重新确定 Team ID
- `OnControllerChangedTeam`：将 Controller 的 Team 变更传播至 Character
- 修改 Team ID 需要满足：没有 Controller 约束 + 服务端权限

## 加速同步 (ReplicatedAcceleration)

```cpp
struct FLyraReplicatedAcceleration {
    uint8 AccelXYRadians;   // 水平方向角量化值 [0, 255]
    uint8 AccelXYMagnitude; // 水平大小量化值 [0, 255]
    int8  AccelZ;           // 垂直分量 [-127, 127]
};
```

- `PreReplication`：将 `FVector` 用 Cartesian→Polar 转换为极坐标格式，量化到 uint8/int8
- `OnRep_ReplicatedAcceleration`：Polar→Cartesian 还原后调用 `SetReplicatedAcceleration`
- `DOREPLIFETIME_CONDITION(ReplicatedAcceleration, COND_SimulatedOnly)` — 仅复制给模拟端

## FastShared 复制

- `ULyraReplicationGraph::InitGlobalActorClassSettings` 绑定 `FastSharedReplicationFunc`
- 服务端 `UpdateSharedReplication` 对比 `LastSharedReplication`，有变化时调用 `FastSharedReplication(SharedMovement)` via `NetMulticast, unreliable`
- `FSharedRepMovement` 包含：RepMovement、RepTimeStamp、RepMovementMode、bProxyIsJumpForceApplied、bIsCrouched
- 客户端 `FastSharedReplication_Implementation` 更新 timestamp、movement mode、location/rotation/velocity、jump force、crouch
- `bIsUsingFastSharedReplication` 标记控制，客户端收到非 FastShared 更新时转为全量更新

## 死亡流程

```
OnDeathStarted
  ├── DisableMovementAndCollision()
  │     ├── SetIgnoreMoveInput(true)
  │     ├── SetActorEnableCollision(false)
  │     ├── StopMovementImmediately()
  │     └── DisableMovement()
  └── [延迟] → OnDeathFinished

OnDeathFinished
  └── DestroyDueToDeath()
        ├── K2_OnDeathFinished() — BlueprintImplementableEvent
        └── UninitAndDestroy()
              ├── DetachFromControllerPendingDestroy()
              ├── SetLifeSpan(0.1f)
              ├── ASC 清理
              └── SetActorHiddenInGame(true)
```

- `OnDeathStarted` 禁用移动和碰撞
- `OnDeathFinished` 执行销毁流程，0.1 秒后真正销毁
- `FellOutOfWorld` 调用 `HealthComponent->DamageSelfDestruct(true)` 触发自杀

## 运动模式 Tag 系统

- `MovementModeTagMap` 将 `EMovementMode` 映射到 `FGameplayTag`：Walking、NavWalking、Falling、Swimming、Flying、Custom
- `OnMovementModeChanged` 清除旧 Tag、设置新 Tag（`SetLooseGameplayTagCount`）
- Crouch：`OnStartCrouch` 设置 `Status_Crouching` tag，`OnEndCrouch` 移除
- `CanJumpInternal_Implementation` 去除了父类的 crouch 检查

## 构造函数配置

- `SetDefaultSubobjectClass<ULyraCharacterMovementComponent>` 注入自定义移动组件
- 碰撞 Profile：`NAME_LyraCharacterCollisionProfile_Capsule`、`NAME_LyraCharacterCollisionProfile_Mesh`
- 移动参数：`MaxAcceleration=2400`、`BrakingFriction=6`、`GroundFriction=8`、`RotationRate=720`
- 创建子组件：PawnExtComponent、HealthComponent、CameraComponent
