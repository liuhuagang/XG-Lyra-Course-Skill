# 输入系统

## 概述

Lyra 的输入系统基于 UE5 的 Enhanced Input 体系，通过 `ULyraInputComponent` 和 `ULyraInputConfig` 提供声明式的输入绑定方式，将输入动作映射到 GameplayAbility。

## 核心类

| 类 | 基类 | 职责 |
|----|------|------|
| `ULyraInputComponent` | `UEnhancedInputComponent` | 绑定额外的 InputAction 和 Tag 查询 |
| `ULyraInputConfig` | `UDataAsset` | 输入配置数据资产，定义 Tag→InputAction 映射 |
| `ULyraInputModifiers` | `UInputModifier` | 输入修饰器系列（灵敏度、反转轴等） |
| `ULyraInputUserSettings` | `UEnhancedInputUserSettings` | 用户输入设置（按键映射配置） |
| `ULyraPlayerMappableKeyProfile` | `UEnhancedPlayerMappableKeyProfile` | 按键配置文件 |

## ULyraInputComponent

```cpp
class ULyraInputComponent : public UEnhancedInputComponent
```

- 继承自 `UEnhancedInputComponent`，新增 `ULyraInputConfig` 引用
- 提供 `TArray<uint32> TaggedBindings` 用于追踪按 Tag 绑定的输入
- `AddTaggedBinding`/`RemoveTaggedBinding` 管理绑定生命周期

## ULyraInputConfig

```cpp
UCLASS(BlueprintType)
class ULyraInputConfig : public UDataAsset
{
    GENERATED_BODY()
public:
    UPROPERTY(EditDefaultsOnly, BlueprintReadOnly)
    TArray<FLyraInputAction> AbilityInputActions;

    UPROPERTY(EditDefaultsOnly, BlueprintReadOnly)
    TArray<FLyraInputAction> NativeInputActions;

    // 核心绑定方法
    void BindAbilityActivation(ULyraInputComponent* InputComponent,
                               ULyraAbilitySystemComponent* ASC) const;
};
```

### FLyraInputAction

```cpp
USTRUCT(BlueprintType)
struct FLyraInputAction {
    UPROPERTY(EditDefaultsOnly, BlueprintReadOnly)
    UInputAction* InputAction;      // Enhanced Input Action 资产

    UPROPERTY(EditDefaultsOnly, BlueprintReadOnly, Meta = (Categories = "InputTag"))
    FGameplayTag InputTag;          // 对应的 GameplayTag
};
```

### 绑定流程

```
ULyraHeroComponent::OnSetupInputComponents
  └── ULyraInputConfig::BindAbilityActivation(InputComponent, ASC)

BindAbilityActivation:
  for (FLyraInputAction InputAction : AbilityInputActions)
  {
    FGameplayAbilitySpecHandle AbilityHandle =
        ASC->BindAbilityActivationToInputComponent(
            InputComponent,
            FGameplayAbilityInputBinds(/* 配置 */));

    InputComponent->BindAction(InputAction.InputAction, ETriggerEvent::Triggered,
        /* 回调 */);
  }
```

### 事件类型

- `AbilityInputActions` — 绑定到 GameplayAbility，由 ASC 自动处理激活/取消
- `NativeInputActions` — 绑定到原生 C++ 回调，用于非 GAS 的输入处理

## 输入修饰器

`LyraInputModifiers.h` 中定义了一系列输入修饰器：

| 修饰器 | 功能 |
|--------|------|
| `ULyraSettingBasedScalar` | 基于设置的标量修饰器，从 `ULyraSettingsLocal` 读取灵敏度值 |
| `ULyraInputModifierAimInversion` | 瞄准反转轴修饰器 |
| `ULyraInputModifierGamepadSensitivity` | 手柄灵敏度修饰器 |

## 输入用户设置

- `ULyraInputUserSettings` 继承自 `UEnhancedInputUserSettings`
- 管理 `UInputMappingContext` 的注册和优先级
- 处理 `UEnhancedPlayerMappableKeyProfile` 的按键配置

## 数据资产

- `ULyraAimSensitivityData`：灵敏度预设数据资产，配置不同瞄准类型的倍率

## 典型配置

```
InputConfig (DataAsset)
  ├── AbilityInputActions:
  │     ├── IA_Jump → InputTag.Action.Jump
  │     ├── IA_Fire → InputTag.Action.Fire
  │     ├── IA_Aim  → InputTag.Action.Aim
  │     └── ...
  └── NativeInputActions:
        ├── IA_Escape → InputTag.Action.Escape (打开菜单)
        └── ...
```

## 输入绑定管道总结

```
PlayerController 创建
  → ULyraHeroComponent::OnSetupInputComponents
    → 创建 ULyraInputComponent
    → 读取 PawnData 中的 ULyraInputConfig
    → InputConfig->BindAbilityActivation
      → ASC->BindAbilityActivationToInputComponent (GA 绑定)
      → InputComponent->BindAction (原生绑定)
```
