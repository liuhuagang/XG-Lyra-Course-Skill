# 第三人称相机与穿透处理

## ULyraCameraMode_ThirdPerson

继承自 `ULyraCameraMode`，实现第三人称相机的视角计算和穿透避免。

## 核心参数

```cpp
// 相机相对角色偏移
FVector CameraOffset;           // 空间偏移（相对目标位置）

// 瞄准偏移
FVector AimOffset;              // 瞄准时的额外偏移

// 距离
float TargetArmLength;           // 目标臂长
float ArmOriginLagSpeed;         // 臂原点延迟速度
float ArmOriginLagMaxDistance;   // 臂原点最大延迟距离

// 旋转
FRotator TargetRotation;         // 目标旋转（由输入控制）
bool bUsePawnControlRotation;    // 使用 Pawn 控制旋转

// 穿透避免 (Penetration Avoidance)
bool bDoPenetrationAvoidance;    // 启用穿透避免
float PenetrationAvoidanceLagSpeed;  // 穿透回退延迟速度
```

## UpdateView 计算流程

```
1. 确定相机目标位置（Pivot Target）
   └── Pawn 位置 + AimOffset（如果有瞄准偏移）

2. 计算臂位置
   └── Pivot Target + ArmLength × 旋转方向（反向）

3. 应用延迟（Lag）
   └── ArmOriginLagSpeed 控制臂位置变化的平滑速度
   └── ArmOriginLagMaxDistance 限制最大延迟距离

4. 穿透避免
   └── 从 Pivot Target 到相机位置做碰撞检测
   └── 如果有物体阻挡，将相机向前推

5. 应用 CameraOffset
   └── 在相机局部空间应用偏移

6. 计算 FOV
   └── Pawn 到相机的距离影响 FOV（可选）
```

## 穿透避免（Penetration Avoidance）

### FLyraPenetrationAvoidanceFeeler

```cpp
struct FLyraPenetrationAvoidanceFeeler {
    FRotator AdjustmentRot;     // 检测方向（相对相机朝向的旋转偏移）
    float BackwardsDistance;     // 向后检测的距离
    float ForwardDistance;       // 向前检测的距离（可选）
};
```

- 多个 Feeler 可以配置为不同方向
- 检测过程会尝试多个方向，找到无阻挡的位置
- 使用 `World->SweepMultiByChannel(OutHits, ...)` 进行碰撞检测

### 避免算法

```
1. 从 PivotTarget 到理想相机位置做 Sweep
2. 如果检测到阻挡：
   a. 按 PenetrationAvoidanceFeeler 配置尝试其他方向
   b. 每个方向依次做碰撞检测
   c. 找到第一个无阻挡的方向
   d. 使用 PenetrationAvoidanceLagSpeed 平滑过渡到新的相机位置
3. 如果没有阻挡：
   └── 正常使用计算的相机位置
```

### 摄像头穿透避免的碰撞查询

```
碰撞通道: Camera (ECC_GameTraceChannel2)
响应: 阻挡世界静态/动态几何体，忽略 Pawn 自身
```

## UCameraMode 混合

第三人称相机模式在 CameraModeStack 中通常作为基础层，其他模式（如瞄准模式）在其上叠加：

```
CameraModeStack (基础):
  [0] ULyraCameraMode_ThirdPerson (基础第三人称)
  [1] ULyraCameraMode_Aiming (瞄准放大，叠加在第三人称之上)
```

- 混合时，基础层提供位置/旋转/FOV
- 叠加层在基础层之上进行调整
