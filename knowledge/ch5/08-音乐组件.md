# 音乐组件

## 概述

音乐组件是一个蓝图组件系统，负责游戏中背景音乐的播放和管理。通过 MetaSound 实现动态音乐系统，包括战斗强度控制和 Fire Stinger 事件触发。

## 架构

音乐组件是蓝图类（非 C++），通过 Experience 加载流程附加到 Pawn。

| 类 | 类型 | 说明 |
|----|------|------|
| `B_MusicManagerComponent_Base` | Blueprint | 基础音乐管理组件 |
| `B_MusicManagerComponent_FE` | Blueprint | 主菜单专用变体 |

## B_MusicManagerComponent_Base

### 核心功能

| 功能 | 说明 |
|------|------|
| 背景音乐播放 | 通过 MetaSound 播放循环背景音乐 |
| 战斗强度 | 0~1 浮点值，控制音乐紧张程度 |
| Fire Stinger | 战斗事件触发的短促音效 |

### 战斗强度系统

- 战斗强度值在非战斗状态下向 0 漂移（自然衰减）
- 进入战斗时设置为 1
- 战斗强度值越高，背景音乐音量越低（让位于战斗音效）

### Fire Stinger

- 在战斗事件发生时触发
- 叠加在背景音乐之上播放的短音效

## B_MusicManagerComponent_FE

- 主菜单专用变体
- 修改 "is in menu" MetaSound 参数
- 不支持战斗系统

## MetaSound 集成

音乐组件通过 MetaSound 参数系统控制音乐行为：

| 参数 | 说明 |
|------|------|
| 战斗强度 | 控制背景音乐音量衰减 |
| is in menu | 控制主菜单音乐状态 |

## 数据流

```
Pawn（通过 Experience 附加 Audio Component）
  └── B_MusicManagerComponent_Base
        ├── Tick 更新战斗强度漂移
        ├── Combat 事件 → Fire Stinger
        └── MetaSound 参数更新
              ├── 音量控制
              └── 状态切换
```

## 注意事项

- 完全基于蓝图实现，无 C++ 源文件
- 通过 Experience 的 `AddComponents` 机制附加到 Pawn
- Tick 驱动的 Pawn 数据传递到 MetaSound 参数
